# 论原型图、需求文档规范管理的重要性

原型图、需求文档对于IT产品经理已是耳熟能详家喻户晓，但是有多少团队的管理者能
够真正的重视起来/
站在项目管理的角度来看，原型图、需求文档是一个公司的组织过程资产，如果组织过程
资产不规范或者残缺不全那么项目团队必定会不断暴雷采坑。尤其是当你新接手了一条产
品线，后来你发现这条产品线的原型图和需求文档要么是版本功能残缺不全，要么是不规
范，逻辑乱七八糟无迹可寻，那这时候你的心情（心里瞬间有一万头草泥马奔腾而过）~
所以尤其是高级产品经理或者产品团队负责人一定要做好这项基础而又重要的工作，做好
这项工作可以从以下几点规划入手（作为一个采坑无数五年多的产品老人和大家一起分享
）：
一、原型图、需求文档细节规范
现在的IT互联网企业基本上都是敏捷式开发，之前几十页上百页繁琐的PRD文档早已被
淘汰出局，对于中小型公司大多都是原型图文档（原型图+备注）。

二、原型图、需求文档管理规范
原型图、需求文档如果没有统一的管理规范容易造成丢失；新人接手后无法快速了解产品
和项目，可能需要花大量的时间去重新梳理业务和产品的一些逻辑。下面和大家分享一个
本人之前公司原型图、需求文档以及UI设计稿等统一管理的方案。
在本地安装SVN服务端，然后给团队中产品、UI等岗位开通个人账号，可以在自己工作
的电脑端安装桌面SVN的小乌龟可视化操作程序； 团队管理人员根据项目和产品线的需
要在SVN建立文档目录和子目录； 团队人员完成原型图或需求文档可以通过SVN可视
化操作直接上传到对应的目录里； 团队管理人员定期检查文档是否及时更新或者齐全。
如果产品团队用的原型图工具是Axure的话可以实现多人协同编辑同一份原型图文档；先
在SVN建好团队目录，然后打开Axure点击左上角工具栏的“团队”。


然后点击从当前文件创建团队项目，出现下面弹框。
输入团队目录的链接和团队项目名称后点击创建按钮即可，然后把团队目录链接和团队项
目名称告诉团队相关人员；相关人员要访问或者编辑点击Axure左上角工具栏的“团队-
获取并打开团队项目”。


输入团队目录链接点击获取即可；也可以用Axure Share，这个就不需要本地安装SVN
了，在云端操作的（比较简单，就是服务器在国外有时候操作会报错）。
最后
合理规范的原型图、需求文档好处：
能够减少研发童鞋的学习成本，提高团队协同效率； 能够形成统一规范的公司组织过程
资产，对其他项目或者团队新人来说能够减少不必要的基础工作； 能够形成系统性规范
，对产品新人成长起到指导性的作用。
