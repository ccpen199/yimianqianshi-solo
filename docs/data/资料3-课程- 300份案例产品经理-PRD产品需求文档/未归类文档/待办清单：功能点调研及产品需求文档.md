# 待办清单：功能点调研及产品需求文档

编辑导读：俗话说得好：“好记性不如烂笔头”，当要做的事情很多很复杂时，最好的做
法就是列一个待办清单，这样做事才会井井有条。本文作者将以番茄to do和滴答清单两
款产品为例，对待办清单产品进行分析，希望对你有帮助。
一、背景和目的
通过对待办清单类产品进行用户调研、竟品功能点调研，了解产品业务逻辑和优劣势，产
出调研报告；确立待办清单产品定位，并构思产品业务框架。
二、用户需求分析
待办清单主要用户群体为19-29岁学生及白领人群，解决用户生活备忘、习惯培养、学习
和工作场景中任务规划等问题。


2.1 需求场景
用户1：大二男生小白平时酷爱打游戏，自制力弱。还有一个月期末考试，小白加入番茄
to do的班级自习室，看到同学们的专注排名，有了紧张感。他开启番茄to do的强提醒
模式和学霸模式，到学习时间强制自己学习，并限制手机打开其他应用，强迫自己专注。 
用户2：大四女生敏敏目前准备考研。敏敏学习认真，但学习方法不理想，不能很好地查
漏补缺。她使用番茄to do的待办集规划自己的复习任务，每天使用番茄钟完成一定目标
，并通过数据统计功能查看自己的学习时间分布，合理安排学习计划。 用户3：职场新人
小风，毕业后成为某公司设计师助理。刚入职场的小风没找到方法，工作没有规划，有时
抓不住重点，工作任务无法按时完成。他使用滴答清单记录每日任务，通过优先级排序更
清晰地了解工作计划，并对重要任务设置提醒，避免遗忘或延期。 用户4：职场精英刘驯
工作四年，目前是上海某律师所骨干律师，工作需要严格的时间规划。他开通了滴答清单
的会员版，以每日时间轴的形式清晰规划当日工作，避免时间冲突。使用滴答清单的待办
编辑功能，对单个案件加以文字备注，查看时一目了然。因为经常出差，不便使用电脑时
，滴答清单的多端同步功能，使他可以用手机查看工作。
三、功能点调研
通过前期调研，“番茄钟”和“待办清单”是产品核心功能点，以下就竟品进行功能点调
研，了解产品设计逻辑和优劣势。

调研对象：番茄to do和滴答清单。

do重点功能为番茄钟，滴答清单重点功能为待办清单，以此两款产品分析对比“待办”
和“番茄钟”功能的设计逻辑和优劣势。

3.5 功能点调研总结
3.5.1 番茄to do
两款产品定位有所不同，番茄to do用户更偏向于学生和年轻白领等自制力较弱、进行习
惯培养的人群，在待办模块使用卡片式陈列，美观度高，但不便修改，不能补充文字描述
，适合于长期稳定的、重复性任务，如背单词、运动等；待办与番茄钟高度关联，能解决
用户自制力不足的问题，数据统计功能强大。


3.5.2 滴答清单
滴答清单更适合自制力强，有明确目标感，待办任务灵活性高、变动频繁，需要进行时间
规划的人群。待办编辑功能强大，能对优先级、子任务、附件、图片、模板等进行编辑，
满足用户文档编辑的需要；提供搜索功能，便于待办管理。与日历高度关联，日程规划一
目了然。番茄钟模块可快速开始专注。


四、产品定位
4.1 产品定位和主要功能
经过前期用户调研及产品功能调研，进行产品定位如下。
用户人群：主要为学生，其次为白领人群。 解决问题：解决学生和白领人群生活场景下
生活备忘、习惯培养等问题；解决学生群体没有学习动力、自制力不足，以及制定科学学
习计划的问题。 产品定位：主要面向学生群体和白领人群，清晰区分长期计划和每日清
单，帮你科学进行时间管理；开启专注模式，拒绝分心不做手机党；与学霸好友建立自习
室，督促自己完成逆袭。 主要功能点：番茄钟、清单、计划、统计、自习室。
4.2 产品基础逻辑
产品以番茄钟为主要功能，待办清单为辅要功能。强调习惯培养的同时，满足用户记录备
忘事项、查看课程表等日程管理需求。
待办清单帮助用户更有计划地使用番茄钟，并满足日常备忘需要； 番茄钟为自制力不高
人群提供专注功能，设置固定的专注任务清单，避免计划性不强的用户无从开始； 数据
统计带来正向反馈，鼓励用户坚持目标。 根据产品上线后数据表现，考虑在之后的版本
加入勋章体系和自习室，鼓励用户坚持目标。

标性不强的人群提供固定番茄钟列表，帮助用户快速进入专注状态。

指灵活变动的日程管理类任务，如生活备忘、工作计划等； 以待办清单的形式呈现，关

八、总结
通过用户需求分析和功能点调研，完成产品猫爪待办设计。产品主要面向学生及白领用户
，解决用户生活备忘、习惯培养以及学习场景下动力不足、自制力不足的问题，主要功能
点包含番茄钟、待办清单、数据统计，后期可能加入自习室及勋章体系。
相比于同类型竟品，根据用户实际使用场景，将待办模块区分为“计划”和“待办清单”
两类，分别满足长期固定的习惯培养类任务和灵活变动的日程规划任务；通过优化计划和
待办清单模块的联动，帮助用户快速进入专注场景。
