	•	文档介绍

文档介绍是向该文档的读者对象进行的介绍说明。它是针对该文档的定义与注释，而非产品。目的是为了方便读者对象更加清楚的读懂该文档。包括以下内容：

	•	文档目的

说明该文档写作的原因及希望达成的目标。

	•	读者对象

说明阅读该文档的人群，如产品经理、设计人员、技术人员等。

	•	名词解释

针对产品需求文档出现的特定名词进行解释说明。

	•	产品概述

产品概述是对产品的简单介绍。包括对产品的基本介绍，使用产品目标人群的描述以及该产品市场现状的分析。具体内容如下：

	•	产品简介

对产品的功能、特征及实现的效果进行说明。

	•	用户群体定位

对产品针对的目标人群进行定位。

	•	针对学科

对产品针对的学科进行定位，并说明学科特点。

	•	同类型产品分析

对市场上流行的同类产品进行分析，描述出同类产品中主要竞争对手有哪些，各自的产品优势是什么，我们的产品与对方产品有何差异，优缺点在哪里。


	•	产品功能性需求

	•	业务流程图

此处为产品业务流程图，如果太大可按照功能进行拆分。


	•	前置条件

对用户使用此业务流程的前置条件进行说明，例如是否需要联网、是否需要注册等。

	•	功能模块划分

对拆分出来的功能模块进行文字说明。
	•	必需的功能需求清单

对各功能模块的技术需求进行详细描述。包括内容有：模块名称
