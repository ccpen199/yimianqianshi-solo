# 106短信MRD市场需求文档

建谷·短信宝
市 场 需 求 文 档
2014-04-17
1、文档介绍
本文档主要说明建谷短信宝系统的市场需求，用户分析，和功能分析。 1.1 文档目的
通过本文档，总结分析市场和用户的需求，整理功能设置，为建谷短信宝产品原型设计做指导。 2、市场问题和机会 2.1 市场问题
短信行业在中国通过一二十年的发展，以其特有的优势，和众多的用户支持，规模发展非常大。但在微信、QQ、陌陌等即时通讯软件的冲击之下，短信作为最简单的通讯方式的作用被大大减弱，市场日益萎缩。
2.2 市场机会
从最初点对点的消息传达工具，到如今短信行业与电子商务等互联网行业的结合，有力的推动了短信行业的发展。自近五年以来，触发类短信得到迅猛发展，成为短信行业中的朝阳产业。
2.3 产品问题和机会
传统短信发送市场江河日下，在注入互联网的新鲜血液后，虽然得到很大发展，但也存在不少问题。
目前短信业的主营业务，主要分为两类。一类是触发类短信，另外一类为营销短信。
各种短信产品百花齐放由于网络监管，基础设施建设等因素，产品质量良莠不齐。尤其是营销类的产品，在没有足够的监管之下，日益泛滥，导致整个用户群体对短信行业产生抵触情绪。
触发类短信却紧紧地和互联网联系起来，成为短信行业的发展尖兵。而且网络扩张比传统方式更加迅速，可以呈几何倍数增长，发展潜力巨大。 2.4 技术问题和机会
网络技术发展到今天，技术实现方式已经多种多样，各种技术难题也在逐步解决，网络基础应用技术已经非常成熟。同时我们也有实力雄厚的技术团队，非常成熟的产品和平台。 3、市场概述 3.1 目标市场描述
建谷短信宝的目标市场为整个互联网短信发送市场。 3.1.1 目标市场特征
物联网短信发送市场具有种类广，行业多，产品种类丰富，涉及面广等特点 3.1.2 目标市场趋势
物联网短信市场的发展趋于多样性，触发类短信独领风骚，各类行业，短信产品遍地开花，各类新颖创意层出不穷。而且随着国内经济发展和相关政策的推动，短信行业的发展已经越来越快。 4、目标客户 4.1 目标客户描述
建谷短信宝的客户群为所有有短信发送需求的企业及个人。 4.1.1 目标客户细分
目标客户主要分为企业客户。企业客户用可以分为中型企业和大型企业，中型企业是建谷短信宝产品初期的主要目标客户。大型企业用户为短信宝的长远发展目标客户。 4.1.2 客户动机
中型企业客户：希望通过相关服务，提升企业形象，促进企业发展。同时可以通过短信推送的方式扩展公司业务。
大型企业客户：通过短信推广企业，提升企业和品牌知名度，促进产品销售，增加企业和用户间交流，确保用户身份验证安全。
4.1.3 影响因素
影响客户使用建谷短信宝产品的因素有：已有短信产品，对短信行业的认知，互联网应用知识程度，使用费用，到达率，送达时间等。 4.1.4 客户目标
客户的主要目标有
（1）中型企业
促进各类短信的发送，增加客户数量。
（2）大型企业
宣传企业文化，推广企业产品，提升品牌价值，增加网络账号安全性。
 5、使用者和用户原型 5.1 目标使用者
建谷短信宝的目标使用者为中型企业和大型企业。主要可以分为三类：各种电商类网站、企业门户网站、企事业内部使用。
5.2 原型特征
5.2.1各种电商类网站
此类用户为各种专业从事商业行为的企业，有雄厚的实力进行商业推广和营销，用户注册量非常大，并配有程序开发人员和熟练操作计算机技术人员。
5.2.2企业门户网站
此类用户对从事行业以及该行业产品都有相当的了解，并有基本的计算机操作人员。用户量较大。
5.2.3企事业内部使用
此类用户人员结构比较复杂，一般没有程序开发人员，可能未配备计算机操作人员。 5.3 现实需要
5.3.1各种电商类网站
此类用户需要专业的开发人员，对于计算机的专业化程度要求高。需要各个环节的实际功能，如注册验证码发送，智能提醒，手机优惠券等。
5.3.2企业门户网站
此类用户对于注册验证码和手机优惠券使用较多，可能需要专业接口对接协助。
5.3.3企事业内部使用
这类人群共性基于短信通知的基本功能，但由于这类用户的复杂性，是这类用户需要更加个性的系统支持，如页面展示帮助，接口对接协助，平台使用操作等等 6、市场需求 6.1 功能分类
6.1.1短信宝功能分类
（1）账户注册
此分类主要为账户注册功能。
（2）权限管理
此分类为短信宝通管理员及对应权限的管理功能，主要功能如下：
    添加管理员：添加一个新的建谷短信宝管理员
    删除管理员：删除一个建谷短信宝管理员
    编辑管理员：编辑管理员信息
    管理员权限设置：设置管理员管理权限
（3）产品或服务
  产品或服务主要功能如下：
    SDK开发文档：支持大部分开发语言的API开发包及SDK开发文档。
    短信平台：用户可借助短信平台管理自己的账户，测试接口，查看各项记录等。
    在线支付功能：用户支付使用费用。

6.1.2短信管理功能分类
（1）模板管理
模板管理功能，包括新建模板，模板审核，模板删除，模板调用，具体功能如下：
    新建模板：新建一个全新的模板格式，包含文字内容，参数内容和用户签名。用户签名为必须存在的部分。
    模板审核：提交新建的模板，通知管理员审核。管理员经过检验内容、格式和签名，予以通过或者不通过。
模板删除：对于不符合规定的模板，予以删除。
模板调用：已经审核通过的模板，通过模板ID来调用。
（2）通道管理
此类功能主要对发送通道统一管理，主要功能如下：
    新建通道：添加一个可用通讯通道。
    通道删除：除去过期或者故障的通讯通道。
通道切换：给用户更换故障通道或者提升服务，升级优质通道。
（3）关键词管理
此功能主要管理关键词相关，主要功能如下：
    添加关键词：添加新关键词到关键词数据库。
    删除关键词：从数据库中删除某些关键词。  
（4）会员管理
会员管理主要对会员信息进行管理，主要功能如下：
    新建会员：添加新的用户到会员数据库，配置相关权限。
    编辑会员：编辑修改会员相关信息。
    禁用会员：会员账号作废时，禁用改账号。
（5）记录管理
此分类集中短信的报表统计功能，主要功能如下：
    短信发送历史记录：历史短信数据统计及输出。
    充值历史记录：历史充值数据统计及输出。
