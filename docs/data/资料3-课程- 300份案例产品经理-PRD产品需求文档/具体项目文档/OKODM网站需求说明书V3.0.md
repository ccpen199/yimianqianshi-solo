项目概述
	•	产品介绍
OKODM云智造闪现平台，主要以体现平台品牌，平台文化，产品及服务宣传、活动对接窗口。平台为需求用户提供专业的项目负责人和供应方，全方位的满足客户的需求。
OKODM平台网站，立足于正集源集团近30年的电子产品生产研发、行业产业链资源，旨在帮助需求客户找到问题解决方案，供应方客户找到订单，项目责任人客户找到客户和生产资源。
平台定位：全球智能终端云智造闪现平台
平台价值: 
	•	整合加工制造业上中下资源。打通上、中、下游资源及与终端用户信息不对称、创新不足的痛点 
	•	推广渠道。为平台成品、样品提供展会、展台、新闻发布会 
	•	行业优势。为设计、样品、成品提供专利、认证、知识产权、测试等服务

平台主业务流程

平台主业务流程说明
整体流程是描述网站系统的主要业务过程，主要包含了用户的工业流程、网购(消费)流程。下图蓝色部分为平台工业流程(主流程)，编号为10，11的为网购(消费)流程，其他为分支流程。
      工业主流程说明：
	•	客户在完成完注册，登录，实名认证，签约(线上支付成为签约会员，线下补寄字纸合同)。  签约会员才能使用平台高级功能。如接单服务等。    
	•	客户选择平台业务服务类型，平台服务类型分为：项目责任人方式和供应方企业方式。用户单项选择一种方式。
	•	发布需求表单，平台针对部分类型表单进行审核，并提供建议书，见附件《需求建议书》。审核通过，直接自动匹配项目负责人或供应方。反之，则通知客户修改或调整需求内容。
	•	如果用户选择供应方/项目负责人，则供应方/项目负责人在接到报价任务后，倘若接受任务，则进行报价。反之，则系统自动匹配到下一家供应方/项目负责人。如果没有匹配到任一家供应方/项目负责人，则按用户需求进行公开与不公开处理。如果公开处理，则以公开招标的方式进行，如果不公开处理，则有平台工作人员处理。
	•	待所有供应方/项目负责人报价成功后，统一反馈给客户，供客户自行选择。
	•	客户选择报价后，生成工业订单。
	•	订单生成后，客户可以选择线上或线下支付方式。目前线上支付仅支持现金账户余额支付。客户需先充值金额到自己对应的现金账户。在结单前，平台支持多笔支付记录的情况。
	•	供应方/项目负责人和需求方在定单未结单环节，都可以发起退单，线上只做信息流和通过线上支付的资金流退款的情况。退单时，平台只接受线上支付金额的退款，线下需求方和供应方/项目负责人支付的现金、期票等，平台一概不付责任。
	•	顺利生产后，供应方通过平台确定全部发货，系统通知需求方确认收货(全部)。
	•	需求方确认收到全部货物，并没有发起退单情况，供应方/项目负责人在确认完收到全部款项后，在系统上标记确认收款。该订单进入结单状态。需求方和供应方/项目负责人需要双方互评。流程结束。
网购(消费)业务流程说明：
	•	客户在完成注册，登录，实名认证，签约(线上支付成为签约会员，线下补寄字纸合同)。  
	•	卖方客户首先需要上传发布不同品类的成品、设计作品和服务类产品（认证/专利/知识产权/测试服务）。平台对其发布的产品进行审核，审核通过后，该产品上架，展示在产品区域。
	•	买方客户进入平台后，选择、搜索产品。
	•	对选择、搜索的产品进行支付。
	•	如果是设计作品则用户直接下载即可。如果是成品，则卖方发货，需求方收货确认。如果是服务类产品，则有卖方线下提供服务。服务完后，由客户确认服务结束。
	•	最后，双方互评，订单结束。

产品管理主要包括产品发布、前端搜索功能
	•	流程图


	•	流程说明
	•	签约企业发布成品产品和专利产品。签约个人只能发布设计类产品。
	•	系统按用户选择的产品类型及产品名称，调用对应的专业产品属性表（此属性表由管理员后台设置），若系统暂无此类产品，系统调用通用属性信息表供用户填写。
	•	用户按产品属性表填写对应内容，提交发布。
	•	平台对用户的信息进行质量标准审核，并给出审核状态信息。审核通过后，产品在前台展示，反之，则用户继续修改产品提交，直到产品通过平台审核为止。流程结束。
	•	需求说明
产品管理主要包含产品的维护和前端查询搜索规则。产品维护主要包括产品发布，上架，下架。
	•	发布。商家用户将按需进行发布产品。提交给平台审核。
	•	上架。审核通过的产品，在客户前端进行展示。上架后，用户可以进行正常的网购行为。当出现库存不足时，客户不能购买，只能进行关注产品。
	•	下架。当商家用户没有此类产品时，或不遵守平台运营规则时，则需将产品下架。下架后，客户不能进行正常的网购行为。
整个产品分为成品产品、设计产品、专利产品。成品产品和专利产品的网购流程一致。具体在网购流程里体现。上架的设计产品，在客户支付成功后，供客户自行下载。可一次支付，多次下载(平台可配置不超过5次)。所有上架产品需要支持
	•	二维码分享，线下二维码扫描，跳转至产品详细界面
	•	第三方分享，方便用户快速分享到第三方渠道。如微信，QQ、微博等。至少三个家分享
	•	收藏/关注，方便用户下次查找。

需求管理
需求管理主要包括需求发布，平台审核，供应方/项目负责人匹配，供应方/项目负责人报价，招标管理。
	•	流程图：

	•	流程说明
	•	用户选择产品和需求类型，系统调用对应的专业表单，设置需求信息是否对外公开，设置对外公开时，需求在未匹配到供应方/项目负责人的情况下，显示在公开招标栏。填写完需求发布信息，点击提交，即成功发布需求信息，待平台审核（注：需求信息需平台审核，是否需要审核由管理员后台设置决定）。
	•	平台评审员对需求进行审核，并出需求建议书(见附件《需求建议书》)，审核通过与否，系统都发“系统信息”提醒用户，用户可登陆平台对审核“未通过的需求信息”进入编辑。
	•	审核通过的需求，平台自动配对供应方/项目负责人，匹配到的供应方/项目负责人可选择参与报价。 
	•	对于未匹配到的需求信息，平台判断按用户设置的方式显示，设置公开招标的需求信息，系统前台显示，否则提醒平台管理员；
