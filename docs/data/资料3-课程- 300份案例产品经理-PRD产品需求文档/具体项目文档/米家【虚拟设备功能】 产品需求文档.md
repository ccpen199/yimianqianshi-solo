# 米家【虚拟设备功能】 产品需求文档

作为中国智能家居的先锋，且拥有本地智能家居用户量最大的平台，米家若要继续增加用
户以及设备体量，需要更加创新的交互体验以及推广方法。本文是关于米家【虚拟设备功
能】 产品需求文档，enjoy~
1. 前言
1.1 需求背景
米家的竞品已经有许多添加了【虚拟体验】功能，但体验并不太好，没有发挥使用户感受
到虚拟设备的便利，以及产生进一步购买想法的作用。作为中国智能家居的先锋，且拥有
本地智能家居用户量最大的平台，米家若要继续增加用户以及设备体量，需要更加创新的
交互体验以及推广方法。
1.2 功能简介
用户在没有实际产品的情况下，可以虚拟体验该产品的部分特性，并且可以身临其境地感
受该产品的智能联动场景，培养用户对该设备的兴趣以及对智能场景的体会。
1.3 项目目标
假设新版本用户一周内点击添加按钮的次数为20万，点击添加虚拟设备的次数为4万，
即添加虚拟设备次数占总添加设备点击量的20%。


假设新版本用户一周内新添智能场景次数为10万，其中包括了虚拟设备的次数为2万，
即包括虚拟设备的新建智能任务占新增智能任务的20%。
假设更新版本前一周该虚拟设备对应设备在小米有品的PV为10万，更新后一周该设备
的PV为11万，其中通过虚拟设备PV为1万，即相比版本更新前PV提高10%。
假设旧版本用户平均使用时长为10分钟，更新新版本的用户平均使用时长增加到11分
钟，即平均增加了一分钟，相比未更新前的使用时长提高了10%。

2.4 添加虚拟设备

首次打开米家
正常逻辑：
首次更新完成打开米家时，首页亮度整体调低，显示图示文字和提示点击位置； 当点击
添加按钮时进入设备列表界面；点击×或“我知道了”，则返回正常首页。

设备添加列表
正常逻辑：
虚拟设备分类排在最上方，在新品分类上方； 点击虚拟设备进入虚拟设备介绍页； 初版
仅设置小米米家智能门锁、小米空气净化器3、米家扫地机器人1S三个虚拟设备，分别
对应有品的小米米家智能门锁碳素黑、米家扫地机器人1s、米家空气净化器3。

虚拟设备添加界面
正常逻辑：
当用户启用了智能场景推荐后即默认启用了设备，点击【启用】后跳转至虚拟设备界面； 
智能场景介绍以图片+文字的形式展现，文字在图片的表层上方，每幅图的像素大小固定
400*500，并排往右，用户可往右划选择智能场景，点击任意一个都跳转至对应实际设备
的推荐场景，后续操作和正常设备操作流程一致； 若用户未添加虚拟智能场景，也未点
击【启用】，则算未添加虚拟设备； 添加了的虚拟设备按顺序排在已安装设备的前面，
以浅绿色为背景区分开普通设备。
异常逻辑：
若添加时出现网络问题，弹出非阻断toast提示，持续2秒：“网络好像开小差了，请检
查一下你的网络哦~”； 若更新后首次打开时意外退出，下次打开仍会弹出提示信息，直
到用户点击添加按钮、×或“我知道了”才后续不弹出提示。
边界逻辑：每种虚拟设备仅允许添加一次，若用户二次添加，弹出非阻断toast提示，持
续3秒：“虚拟设备暂仅支持添加一次哦”。
2.5 使用虚拟设备
2.5.1 直接使用虚拟设备

虚拟设备界面
正常逻辑：
从我的设备中点击虚拟设备，启动设备； 展示相应动画以及后续触发联动动画（若有）
，动画下方页面用毛玻璃效果； 播放过程中单击动画外的界面则会提示：“再次点击退
出观看”，退出动画后，返回毛玻璃之前效果；而动画播放完全后仅点击一次就退出动画
，结束后显示该设备模拟运行结果，其余关联场景设备运行结果同正常流程一样在米家
TAB列表显示； 动画可单击视频内任意位置即可暂停，再次点击即恢复播放，单个动画
时长控制在3s内； 不显示动画进度条及音量选项，仅展示动画，暂停后画面显示为暂停
时的动画和播放按钮，点击播放后，播放按钮在0.5s内渐变消失。

虚拟设备选项列表
正常逻辑：
在选项列表中，放置【购买本产品】的选项； 包括智能在内的其他功能都和普通设备一
致可调。
异常逻辑：若使用时出现网络问题，弹出非阻断toast提示，持续2秒：“网络好像开小
差了，请检查一下你的网络哦~”
边界逻辑（无）
2.5.2 添加虚拟设备至智能场景中

智能场景选择条件
正常逻辑：
自建智能场景的添加均和正常无区别，虚拟设备在外观上会在旁边备注[虚拟]，如图所示
。

推荐智能场景
正常逻辑：
推荐智能场景中，缺少的设备点击提示进入添加虚拟设备的界面； 添加完成后不跳转到
虚拟设备界面，返回该智能场景设置界面； 点击启用后，即把本智能添加到【我的】模
块中； 智能场景指令中，虚拟设备皆以“设备名称”[虚拟]的样式显示名称。
异常逻辑：若添加时出现网络问题，弹出非阻断toast提示，持续2秒：“网络好像开小
差了，请检查一下你的网络哦~”
边界逻辑：推荐智能场景中，添加一个设备即可启用，现在规则为至少需要两个设备才可
启用。
2.5.3 智能场景触发
2.5.3.1 手动触发

手动触发场景
正常逻辑：
智能场景从列表上看，虚拟设备图片下方用绿色的点以标志； 点击执行后，实际设备照
常运行，其中的虚拟设备以动画形式展示，动画下方页面用毛玻璃效果，同直接运行模拟
设备一致； 播放过程中单击动画外的界面则提示：“再次点击退出观看”，退出动画后
，返回毛玻璃之前效果；而动画播放完全后仅点击一次就退出动画，退出后的虚拟设备运
行完成状态在米家TAB的设备列表中显示； 动画可单击视频内任意位置即可暂停，再次
点击即恢复播放，单个动画时长控制在3s内，按虚拟设备的顺序播放动画； 不显示动画
进度条和音量，暂停后画面显示为暂停时的动画和播放按钮，点击播放按钮在0.5s内渐
变消失。
2.5.3.2 自动触发

自动触发场景
正常逻辑：
设备触发、定时触发或语音触发皆对虚拟设备有效，正常设备正常运行，模拟设备模拟运
行； 虚拟设备模拟工作时在如图位置显示工作中，可点击暂停按钮使设备结束模拟运行
； 点击进入模拟设备，同直接运行模拟设备一样显示动画，动画结束则运行完成，模拟
设备的可暂停状态变为可启动状态； 若用户在智能场景中用了虚拟设备后一直没有点击
进入观看，则一直显示可暂停，即运行状态。
异常逻辑：若模拟设备显示在模拟运行中没有停止，用户直接或间接地施加了新的操作，
则以用户的新操作为显示准则。
边界逻辑（无）
2.6 删除虚拟设备
正常逻辑：虚拟设备的删除和智能场景中虚拟设备的删除都同正常设备一样操作。
异常逻辑：若删除时出现网络问题，弹出非阻断toast提示，持续2秒：“网络好像开小
差了，请检查一下你的网络哦~”
边界逻辑（无）
2.7 其他
个别虚拟设备因为操作不依赖手机上直接选择，需要一些特殊交互，比如指纹锁的开门操
作。

点击模拟开锁后出现指纹图案，大小和绿色圆圈一致； 将在该区域按压的操作模拟成指
纹开锁的操作； 其他流程同直接使用虚拟设备一致。
异常逻辑：
当用户触碰区域不在图片指纹范围内时，弹出绿色提醒文字：“请将手指放在识别区域内
”，识别成功后绿色文字消失，正常显示运行结果； 若操作时出现网络问题，弹出非阻
断toast提示，持续2秒：“网络好像开小差了，请检查一下你的网络哦~”
边界逻辑（无）
