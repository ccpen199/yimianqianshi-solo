# 倒推手机淘宝App需求文档PRD

本文根据淘宝app的底导航中五个主要模块以及从浏览商品到订单提交做了一个需求分
析文档。包括：文档综述、需求整理、需求结构、全局说明、主要功能说明。
手机淘宝作为目前最大的电商平台，打败了线下实体商店，给了广大群众最方便快捷的购
物体验。
人们不用出门就可以买到想要的并且比实体店实惠很多的东西，用户想买的东西在淘宝都
能找到，比如衣服、母婴用品、电子产品、化妆品等；想不到的东西在淘宝也能找到，比
如教育视频、代写、专家指导、农村特产等。
不论吃的喝的穿的用的玩的学的都可以在这个大平台里满足你。
由于目前淘宝功能模块太多，此次我通过体验和分析这款app，根据淘宝app的底导航
中五个主要模块以及从浏览商品到订单提交做一个简单的需求分析文档。

四、全局说明
1. 功能权限
4.1.1 未登录状态下


不能加入购物车、购买商品；无法客服聊天、收藏、查看微淘上新、我的关注。 不能进
入“我的”、“消息”、“购物车”、“会员码”页面，点击直接进入登录页面。
进行以上操作都会转跳到登录页。
4.1.2 登录状态下
可进行任何操作
2. 键盘权限
点击手机注册或者手机登录时，输入框时弹出数字键盘； 点击账号密码登录、搜索框、
评论框、聊天框时⻚面底部弹出字母全键盘。
3. 页面内交互
4.3.1 底导航切换
位置：底导航固定在手机页面最底部。
交互：
用户在点击“我的”、“微淘”、“消息”、“购物车”、“我的淘宝”进入到对应的模
块中。 选中的按钮文字变为红色。
4.3.2 键盘弹出
位置：弹窗固定在手机页面底部。
交互：
点击【搜索框】，页面底部交互页面变暗，弹窗由底部向上弹出。 点击【取消】回到上
级页面。
4.3.3 商品搜索
位置：搜索框位于页面导航栏中间。
交互：

当用户点击【搜索框】时，弹出键盘，点击【完成】后进入“搜索”页面。 点击【返回
】返回上级页面。 点击【取消】搜索页面由上至下消失，父级页面淘宝主页显示出来。
4.3.4 导航栏显示和定位
位置：导航栏随着用户对页面的滑动显示和隐藏在页面的固定位置。
交互：
当用户向上滑动屏幕时，导航栏逐渐显示。 页面滑动或用户滑动页面至顶部时，导航栏
逐渐隐藏。 用户点击导航栏按钮时，页面滑动至对应区域，下划线滑动至导航栏对应按
钮。
4.3.5 详情页分享
位置：分享位于导航栏右上角，点击分享弹窗位于手机底部。
交互：
点击分享底部弹出分享列表框，用户可以分享至多个平台或者淘宝好友。 图片分享逐渐
弹出，用户可以下载分享至其他的平台或好友。 点击暗部区域或者【取消按钮】，分享
弹出框逐渐隐藏。
4.3.6 选择规格加入购物车成功
位置：规格弹出有底部向上弹出，固定于手机底部。
交互：
当不需要选择规格时，直接提示添加成功。 当用户点击【加入购物车】后出现规格选择
，页面底部交互页面变暗，弹窗由底部向上弹出。 选择规格完成，点击【确定】按钮确
认添加至购物车。 用户点击【确定】后，规格弹出隐藏，提示用户添加成功。
4.3.7 下订单
位置：下订单按钮，固定于手机底部
交互：

点击购物车底导航进入购物车页面进行商品购买操作。 当用户点击单选框或者全选框点
击的单选框变为选中状态，如果该店铺被加入购物车的商品只有一个，那么该店铺也变为
选中状态，意思就是提示用户该店铺的商品已经被全部选中。 用户可以查看到结算的数
量和价格。 选择完成后，点击【结算】按钮进入至确认订单页面。

五、主要功能说明
1. 登录注册
5.1.1 页面名称
登录页面、注册页面
5.1.2 页面入口
当用户未登录时，点击“消息”、“购物车”，“我的淘宝”进入登录页面，在主页中，
点击【立即登录】、【我的会员码图标】、快捷按钮进入登录页面。在微淘模块中，点击
【上新】、【关注】、【我的关注】、【登录】进入登录页面。用户点击立即注册进入注
册页面。
当用户登录成功后，不再出现注册登录页面，用户可以在设置中退出登录重新进入登录页
面。
5.1.3 页面功能
手机号码、邮箱、用户名登录，支付宝快捷登录、找回会员名、手机号码注册。
5.1.4 页面逻辑


登录页面可以提供用户手机号码、邮箱、账户名以及快捷方式登录。 当用户填写好手机
号码、邮箱或账户名时，登录按钮变亮即变为可点击状态。 用户可以点击更多选项找回
会员名或寻求帮助。 用户不登录时可以正常查看淘宝主页，微淘主页、商品信息、图片
、分享、扫一扫。 当用户想进行购买、加入购物车、查看我的、消息、上新和关注、评
论、查看会员码时需跳转至登录页面登录。 用户注册时，点击手机号码区域选择可进行
区号更换。 当用户开始填写好手机号码时，注册按钮变亮即变为可点击状态。 当用户输
入的数字超过11位时，或不符合后台规则时，按钮至灰。 用户输入正确，跳转至验证码
输入页面并开始倒计时，超过一分钟则重新验证。 输入验证码页面中，用户点击返回可
返回上级页面。
2. 首页
5.2.1 页面名称


淘宝首页
5.2.2 页面入口
用户打开手机淘宝App时，出现的页面，用户也可点击底导航进入首页。
5.2.3 页面功能
扫一扫、搜索、快捷按钮、商品展示。
5.2.4 页面逻辑
首页提供搜素、扫一扫、会员码、快捷按钮入口、商品展示。 搜索框的默认关键字根据
用户的购买记录浏览记录更换。 用户上滑下滑均可进行刷新。 当用户上滑时，底导航首
页按钮更换，点击可以直接进入最顶部。 当用户上滑至商品展示列表区域时，展示分类
栏至于顶部。 扫一扫可以切换AR扫码或拍立淘。 搜索页面同输入框一同显示，当用户
返回至搜索页面或进入搜索页面时，输入键盘一同弹出。 商品分类模块中，用户左滑可
以看到更多商品分类。

3. 商品详情页面
5.3.1 页面名称
商品详情页面
5.3.2 页面入口
用户在首页、微淘、订单等模块点击商品item，或者搜索结果列表进入。
5.3.3 页面功能
搜索、分享、购物车、收藏、联系客服、加入购物车、立即购买。
5.3.4 页面逻辑


详情页面是一件商品的详细展示，包含了价格、标题、发货地、商品规格、评价、问答以
及详情和推荐方便用户查阅。 用户可以返回上级页面、分享给淘宝朋友或者分享到其他
平台。 当用户点击店铺时，进入卖家店铺。 当用户点击客服时进入客服聊天窗口。 点击
更多由上至下弹出选择框，用户可以点击按钮进入其他页面比如“消息”、“首页”。 
点击加入购物车弹出规格选择框用户选择后提示加入成功，点击立即购买弹出规格选择框
用户选择后进入订单提交页面。 用户点击图片进入图片全屏页面，长按可保存图片或查
看原图，单击图片可退出全屏，点击【取消】返回详情页面。 点击分享图标时，分享列
表有底部弹出，可以分享至微信、朋友圈、QQ、钉钉、支付宝和短信。可以操作复制链
接等其他操作。 当用户点击图片右上角的下载按钮，或长按图片可以保存图片至本地相
册。
4. 购物车页面
5.4.1 页面名称
购物车页面
5.4.2 页面入口
用户在底导航点击购物车模块、商品详情中点击购物车图标进入购物车页面。
5.4.3 页面功能
管理删除和清理、选择结算、商品查看、数量加减。


5.4.4 页面逻辑
购车页面展示了喜欢商品的列表。 用户可以点击管理、管理购物车的数量对商品进行移
除和移入收藏夹，用户也可以对某个商品右滑进行一样的操作。 当用户对商品右滑时，
出现按钮选择框，点击“删除”，删除当前商品。点击“移入收藏夹”可把商品转移至收
藏夹产品中，点击“找相似”可以进入找相似页面。 用户点击店铺名称可以进入店铺主
页、点击商品可以进入商品详情页面，点击规格可以重新选择规格。 商品的数量可以通
过点击加减按钮实现数量的改变 用户可以选择某个商品点击单选框或者直接点击全选，
去结算商品。
5. 订单页面


5.5.1 页面名称
我的订单页面
5.5.2 页面入口
用户在“我的淘宝”里面可以直接点击某个状态的订单进入订单列表，或者点击【查看全
部订单】进入订单列表。
5.5.3 页面功能
搜索订单、删除订单、查看物流、确认收货、追加评价、卖了换钱、申请开票、提醒发货
、加入购物车、其他。
5.5.4 页面逻辑
用户点击查看商品订单时，【全部】按钮选中，用户可下滑查看全部订单。 用户可以点
击其他状态按钮具体查看某个状态的所有订单。 当点击搜索框时，弹出搜索页面用户可
以搜索具体订单，点击【取消】搜索框隐藏。 订单商品可以点击进入状态详情页面，或
是直接在订单列表点击快捷按钮进行操作。 用户已经购买完成的商品，可以直接点击加
入购物车再次购买。
六、总结
手机淘宝功能繁多框架庞大，内嵌的平台也很多。包括近些年火起来的直播也是淘宝的一
大流量来源之一。
作为淘宝的粉丝，我写的还只是冰山一角，而且经验尚且不足请多包涵。在疫情期间，盒
马生鲜、天猫超市、口罩抢购给予了我很大的帮助，以下是我对淘宝的一些看法。
1. 优点
功能zhon各类齐全。用户可以在app里购物、买菜、看电影、看直播、学习穿搭等，还
可以与国际接轨，随时买到世界各地的产品。 购买方便安全。淘宝与支付宝结合增加了
安全性，用户通过第三方平台付款。淘宝拥有自己的物流菜鸟裹裹，包裹送货上门，退换
货也是上门取货，商品大部分都是可以七天无理由退换，用户可以随心所欲的购买，新推
出的88会员还大大降低了购买成品。 分享方便。分享的方式多样化，保证用户可以分享

到各个平台。 页面浏览切换流畅。 扫一扫和图片搜素商品精确度高。 可以帮组家人或者
家人帮助购买商品。 消息群，随时了解到最新福利。
2. 我的意见
很多模块的导航栏无法所有滑动，比如我的订单里面【全部】到【待付款】只能通过点击
。 可以增加一个订单同商品购买分享功能，当用户B想要用户A的某个商品时，用户A
可以把自己的订单分享给用户B，当用户B点击后直接进入到用户B的购物车页面，商品

全都是该时期的，用户点击单选按钮或者全选即可删除。当用户点击右上角的【清除筛选
条件】即可恢复到正常的购物车列表。 关注的店铺或商品分类，用户可以自己建立分类
名称，比如【衣服】、【电子产品】，当用户想去看自己喜欢的店铺或商品时，可以精确
查看某个类别的店铺或商品。这样也可给给用户一种独特的感觉。
