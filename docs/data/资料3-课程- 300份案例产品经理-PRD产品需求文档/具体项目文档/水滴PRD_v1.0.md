1一、文档简介
此文档为”水滴v1.0.0”的产品需求文档，主要用于定义水滴app的产品结构，详细描述各个模块
的内容、功能和相关逻辑。
1.1目的
此文档的主要目的是清晰、有层次的定义产品的各个模块，描述清楚产品各个模块涉及的内容，
功能，以及相关逻辑，作为需求确认以及产品设计的主要依据。
1.2范围
此文档主要描述产品中前端页面的布局，页面呈现的内容和所涉及到的功能点，以及部分交互
细节。本文档主要针对技术部的前端工程师和设计部的视觉设计师。
二、产品概述
一款基于时间管理的习惯养成应用，为生活中的各种习惯提供智能提醒，数据统计等功能，最大
化的满足人们对于习惯养成、时间管理、实现目标所产生的需求，并且为用户带来一个有价值，
可以互相给予鼓励和监督的社区，为用户更好养成习惯的同时为公司带来高粘度的用户。
2.1背景
随着智能手机的普及，人们的时间趋于碎片化，注意力被分割，拖延症严重，迫切需要一款产
品帮助他们养成习惯，管理时间，改变固有生活方式，传递正能量

22.2目标
构建一个国内最好的习惯养成应用与用户交流平台

8Usercase登录
前置条件用户未登录时，可以在社区进行浏览，但需要进行一些和个人账户有关的操作，
如点击圈子、我的、打卡、记录一下等时，弹出登录页
主流程1.输入账号，密码。点击登录后系统验证正确即可完成登录。
2.快捷登录选择微信/微博/QQ登录时，跳转至第三方登录页面，用户允许获取
用户昵称头像等公开信息后自动跳转至水滴
异常流程 1.检测到无网络连接时，提示网络异常
2.点击登录后，数据库中没有检测到被填写手机号的数据时，提示该账户不存在
3.点击登录后，系统验证密码与数据库中账户的密码不符时，提示密码错误
后置条件 提示登入成功，用户可以开始使用圈子、我的等功能

93.1.4注册流程需求说明
说明：
1.用户手机号码验证码均填写正确后，进入个人资料编辑页面
2.用户ID具有唯一性，若用户昵称已存在，提示用户重新填写
3.注册成功后直接跳转至登录成功页面


10Usercase注册
前置条件 用户需要进行登录操作且想注册账号时
主流程1.登录注册页面弹出后，点击注册进入注册页
2.填写手机号码并获取验证码，系统发送验证码至被填写手机号码
3.用户在规定时间内验证码填写正确即可点击下一步按钮
4.填写基本信息（密码、昵称、性别）完成注册
异常流程1.号码格式填写错误，提示用户重新填写
2.号码在数据库中已存在，提示用户直接登入或换个号码
3.验证码填写超时，提示用户重新获取
4.验证码填写错误，提示用户重新填写
5.用户昵称已存在，提示用户重新填写
功能 说明
获取验证码 点击获取验证码后系统发送验证码至被填写的手机号码，验证码有效时长为60
秒，超时需重新发送。验证码发送后60秒之后才能重新获取
设置昵称 用户昵称具有唯一性，若用户昵称已存在，提示用户重新填写
