# PRD：倒推“映客APP”产品需求文档

映客作为直播行业的头部平台，2018年上市。继版本6.0更新后，映客开始重点发力在
社交关系。文章通过使用、体验、研究等方式对映客最新版本进行分析，详细解读其功能
设计逻辑，撰写成本文的PRD文档。

三、全局说明
3.1 功能权限
3.1.1 登录/未登录
登录状态可进行所有操作； 未登录状态下，启动页结束后进入登录页，完成登录之前仅
可进行手机号验证登录、第三方授权登录操作。
3.1.2 普通用户/VIP用户
普通用户可以浏览所有页面执行除VIP专属权益（需充值相关特权）的所有操作； VIP
用户可享受APP内相关专属特权（专属礼物、尊贵座驾、真爱守护团等）。
3.2 键盘交互说明
手机号登陆页面点击手机号和验证码输入框时，由页面底部向上弹出数字键盘； 点击聊
天界面、搜索等输入框时，由页面底部向上弹出字母键盘。

五、页面功能详细说明
5.1 登录页
页面名称：登录
入口：初始登录APP
基本事件流程：
app显示登录表单页面，表单中包含文本：①输入手机号码②输入验证码 用户输入11位
手机号码，获取验证码并输入 点击登录按钮，进行登录 登录成功进入首页
页面逻辑：
登录方式：①手机号码登录②提供第三方登录 点击手机号/验证码登录，输入手机号和验
证码进行登录 使用本机登录用户可点击“本机号码一件安全登录”进行登录，并默认”
同意用户使用协议“并授权映客获得本机号码 登录即代表”同意用户使用协议”
后置条件：登录成功/登录失败重新登录
异常流程：用户输入短信验证码错误，弹出“短信验证码错误”提示文本，2秒后消失，
用户重新操作
页面内交互：


点击手机号、验证码输入框时，数字键盘从底部向上弹出 获取验证码后，按钮内容变为
“60S”并降低灰度开始60秒倒数，60S后按钮内容变回原本内容，恢复原本的灰度。
5.2 首页
页面名称：首页
入口：进入APP-首页
前置条件：①登录 APP②网络良好
页面逻辑：
点击顶部tab栏/左右滑动页面空白区域可切换至地区定位/直播/派对相应页面 点击消息
跳转至消息通知页面 可通过用户昵称和映客号进行搜索 点击直播banner进入直播间 点
击广告活动banner进入相应的活动界面 点击任意频道标签进入直播主题频道页面
刷新机制：
下拉页面进行刷新 进入此页面自动刷新
异常情况：


无网络状态下点击Tab栏标签/下拉页面，tab栏下弹出浮窗“当前网络不可用，请检查
你的网络设置”，点击后跳转至检查网络连接页面，点击“前往设置”跳转至手机设置 
无网络下点击频道标签，跳转至功能页且无加载信息 无网络下点击进入直播间，无加载
信息，页面顶部向下跳出弹窗“似乎已断开与互联网的连接”，浮窗1S后消失
补充说明：
推荐模块根据热门排行榜进行排序； 底部状态栏和顶部状态栏位置固定，中间页面可上
下滑动。
功能描述：直播间
前置条件：用户处于直播页面当中
基本事件流程：
用户点击观看直播 用户通过发送评论、送礼物、连麦、私信和主播互动 用户分享直播间
至第三方平台邀请好友一起观看 用户关注主播 用户录屏
其他流程：用户送专属礼物给主播


用户点击送礼物 后台检测用户是否是首次充值映币并推送活动 特权用户赠送专属礼物/普
通用户不能赠送礼物或充值赠送专属礼物
异常情况：无网络状态下无法查看信息内容
补充说明：
点击屏幕任意区域右划可清除屏幕除直播外的内容； 点击屏幕任意区域上下滑动可切换
不同直播间； 点击右下角关闭icon退出直播间返回上一级界面。
5.3 创建直播
页面名称：创建直播间
入口：打开APP-创建-直播
前置条件：①登录 APP②主播开启直播，用户进入直播间③网络良好
页面逻辑：


通过主播是否开启位置定位更精准推送给用户 点击开友聚邀请微信好友/QQ好友/映客好
友进入直播间 点击开启前置/后置镜头 点击编辑直播主题/内容 点击开启/关闭/调整美颜
功能美化直播内容 分享直播间至第三方平台邀请好友进入直播间
刷新机制：固定页面状态，无刷新机制
异常情况：
无网络状态下无法创建直播，页面顶部向下跳出弹窗“似乎已断开与互联网的连接”，浮
窗1S后消失 直播中无网络状态，页面中部显示网络卡顿 直播中闪退APP，再次进入
APP界面弹出dialog浮窗“欢迎回来，是否继续直播？继续/取消”
后置条件：显示本次直播的数据
补充说明：
直播中点击屏幕任意位置可调整画面； 开启友聚功能直播不予保存； 停留创建直播页面
超过8min自动退出创建页返回上一级界面。

5.4 动态页
页面名称：进入APP-附近-广场
前置条件：①登录 APP②网络良好
页面逻辑：
点击顶部tab栏/左右滑动页面空白区域可切换广场/直播 点击发布视屏/照片/音频/文字
内容至广场 点击活动banner进入相应活动主题 点击热门话题进入相应话题页面 点击用
户头像可进入用户主页查看用户信息 可对用户动态进行点赞/评论/私聊/关注 可分享用户
动态至微信好友/朋友圈/微博/QQ/QQ空间
刷新机制：下拉页面进行刷新
异常情况：
无网络状态下无法查看用户发布的内容 无网络状态下无法下拉刷新


补充说明：底部状态栏和顶部状态栏位置固定，中间页面可上下滑动
5.5 我的
页面名称：进入APP-我的
前置条件：①登录 APP②网络良好
页面逻辑：
点击可编辑个人信息 显示个信息 显示个人关注/好友/粉丝数据 可进入平台功能、个人受
益等常用工具页面 可进入App相关设置
刷新机制：固定页面布局，无刷新状态
异常情况：
无网络状态下进入页面，顶部弹出浮窗“似乎已断开于互联网的连接”1S后消失； 页面
顶部弹出浮窗“当前网络不可用，请检查你的网络装置”，点击后跳转至检查网络连接页
面，点击“前往设置”跳转至手机设置； 无网络状态无法查看我的信息。
补充说明：底部状态栏和顶部状态栏位置固定，中间页面可上下滑动


六、非功能性需求
6.1 性能需求
app启动时间不超过5S app内操作的响应时间不超过3秒，在网络连接超时/弱网情况
下，需给出相应提示并可重新加载 前端实现图片懒加载，节约用户流量，保障页面加载
速度
6.2 可用性需求
兼容性：需要 iOS 8.0/Android 7.0及更高版本系统 信息支持缓存机制，只要加载完成，
网络中断后仍可显示信息
6.3 运营需求
6.3.1 用户运营
签到领取奖励，连续签到奖励价值升级/积分加成。奖励所得积分可以兑换礼物/现金，利
用积分体系来培养用户习惯，提升活跃度 做任务赢积分，通过新手任务和日常任务成就
达成送福利，不仅能够帮助用户熟悉 APP 的业务，同时还能够提升用户的活跃度 用户可
以在APP中进行充值/购买特权，不同等级带有不同特权身份
6.3.2 内容运营
不同直播内容开设不同的直播频道专区，通过“直播+”的形式增加内容丰富性 重点发力
社交关系链：动态社区/附近的人/多人连麦等通过主播 围绕当下热点衍生出热门话题，引
起用户参与讨论
6.3.3 活动运营
新人充值礼包：用户首次充值赠送超值礼包，促使用户消费 VIP充值补给：对二十五天以
内充值过的用户给予低价购的权利，刺激用户再次消费 贵族特权：在有一定等级的用户
中开放专属靓号竞拍，获胜者拥有特定账号使用权，刺激用户充值 小时榜：主播凭借自
己在2个月内直播中得到的总收入排名获得首页专属位展示权 抢热门：主播凭借自己在
24小时内直播中率先达到一定积分获得首页黄金位展示权 礼物争夺战：主播在平台所有
榜单中的每日积分排名前三可获得奖励，若一周内获得周冠军且收集单个礼物总价值超过
一定金额有专属奖励。

七、总结
优点：
界面清晰简洁，逻辑简单，将附近的人和动态直接放在一级模块，突出社交属性。
“首页-附近”是基于LBS查看附近的人直播和动态消息，将主播与观众的关系链转移为
基于地理位置的陌生人社交；另一方面，“首页-关注”是基于已关注的人发的直播和动
态消息的熟人社交。映客在社交关系上重点发力，以主播为连接点构建社交关系链，大力
提高了主播和用户粘性。
待优化：
登录流程中对一些异常考虑不完善，应该添加无网络时的友善提示，引导用户进行正确的
网络设置 首页内容模块清晰简单，但频道直播模块插入首页很鸡肋，用户点进频道需要
的步骤：首页-频道标签-频道-直播，延长操作时间影响用户体验（最新版本已改） 对于
用户特权等级和主播的热门排名没有一个清晰的展示通道，用户无法直观了解热门内容和
自身处于什么状态，运营也可通过一些措施刺激用户
