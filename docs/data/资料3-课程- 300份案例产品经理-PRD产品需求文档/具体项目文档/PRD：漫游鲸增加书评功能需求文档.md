# PRD：漫游鲸增加书评功能需求文档

编辑导语：随着互联网的不断发展，人们的交易方式也不断地增多，二手市场也在不断的
扩大，二手市场中除了常见的一些商品以外，书本也成了二手交易的一部分，并且有专门
的二手书交易平台；本文作者分享了关于漫游鲸增加书评功能需求文档，我们一起来了解
一下。
前段时间将漫游鲸和多抓鱼这两款产品进行了对比，一直觉得漫游鲸可以多增加一个书评
功能，毕竟现在漫游鲸的书费不能提现，很多用户可能会因此流失，增加书评功能或许可
以搭建起用户之间的社交圈，更好的留住用户。


一、产品概述
1. 产品介绍
漫游鲸成立于2018年4月，是一款回收二手书，售卖二手书的APP，主要通过小程序轻
量化快速上线，借助微信社交流量快速传播，是目前较热门的一个二手书交易平台。
2. 产品定位
漫游鲸是采用积分模式的二手书流转市场。采用了“积分策略”，采取原价回收，1.5折
买书，卖书所得并不能用于提现，创新式的将卖的钱转化为在平台上流转的积分，买书的
时候利用书费抵扣书籍原价的85%，剩余的15%用现金支付。
3. 产品特点
漫游鲸与市面上二手流转市场不同的特点有三个：平台“原价”回收旧书、工业化处理二
手书、用户“1.5折”买书，即多了一个“虚拟书费”的自制货币体系。
二、项目背景与需求分析
1. 背景
目前漫游鲸面临的问题是面对爱阅读的文艺青年用户，仅仅的二手图书交易已经不能满足
用户的需求，书费又不能提现，很多用户用完就走，留存率低；我认为可以通过增加写书
评的功能，积累优质内容，搭建起漫游鲸的社区建设，加强用户之间的交流，以此来吸引
更多的用户留在平台内。


2. 目的
目前小程序在买书和卖书等基本功能上已经相对完善，但没有写书评这个功能，用户想要
了解该书的内容只能通过其它小程序或者平台的评论去了解，才能为购买书籍做出参考决
策；而且有的用户在卖书时想要写一些自己的心得，或者推荐优质书给大家看，用户之间
交流书籍的内容。
因此漫游鲸决定新增“写书评”的功能，让用户可直接在小程序看到该书的书评，还可以
与其它用户进行交流，这样除了帮助用户决策提供参考之外，还可以增加用户之间的交流
，增强用户粘性。
3. 用户场景需求分析【用户-场景-需求-现有解决方案】
1）通过以上场景分析，提炼出用户的原始需求：
写书评：用户想发表自己对书的理解，通过写书评让更多读者看到，卖书时也能增加购买
率。 看书评：用户想看大家对这本书的看法，希望从不同角度去看待这本书，也想为买
书提供决策依据。 点赞书评：喜欢某本书，想要与别人互相交流想法。看到独特的见解
想要为他点赞。自己写的书评也想获得别人的点赞，想被别人认可。 评论书评：看到其
他用户的评论非常有同感或者不认同，想要与别人讨论。 管理书评：对自己的书评可进
行删除和修改。

3. 调研总结
多抓鱼对比微信读书的书评功能，微信读书会更加多元化、社区化，评论机制相对较完善
，而且基于微信庞大的用户量，具有得天独厚的先天优势；微信读书更多的是满足用户的
社交需求，用户需求的核心是“读书”而漫游鲸的核心是“卖书”，所以只能借鉴其书评
的流程。
同是以“卖二手书”为核心的多抓鱼的书评功能简洁易懂，写书评的流程也简易，漫游鲸
第一阶段完全可以借鉴其写书评功能。
从以上分析可得出结论：
漫游鲸依托小程序进行二手图书交易，没有读书功能，可以增加基本简单的写书评功能来
提高用户留存率。 多抓鱼是漫游鲸的直接竞品，面对的目标用户相同，可借鉴多抓鱼基
本的书评功能：在书籍详情页设置书评入口、基本的给书评分、写感受、修改、删除。 
可借鉴微信读书的权限设置、评论他人书评，其余功能暂时不需要，而且开发难度大，暂
不考虑。


四、需求目的与功能列表
1. 需求目的
为了提高用户的黏度，漫游鲸决定新增“写书评”功能。
当用户在对某一本书对是否购买进行决策时，可以通过书籍详情页其他用户的评论来作为
参考依据；当用户在浏览书籍时看到之前自己看到过的书并想要留下自己的见解，通过“
写书评”功能可看到评论列表并进行评论；当用户想要与别人进行想法交流，想要与别人
互动时，在书籍详情页的评论列表可看到所有用户的评论，可从中选择感兴趣的评论点赞
或者回复交流。

七、总结
漫游鲸现在改版了，将“个人书架”功能删除，页面排版也相对于以前整洁了许多。


而作为用户与用户交易的桥梁，个人认为更应该建立用户与用户之间的联系，选择买卖二
手书的大部分都是平时爱读书的用户；这些用户的需求肯定不仅仅是买卖书籍，更多的是
想要“以书会友”，而“书评”这个功能正好满足了这些用户交流的需求，既可以增加用
户粘性，又可以满足用户交流的需求。
不过或许漫游鲸也有自己的考量，不做书评也必定有它的道理，也许是因为增加书评很可
能会有某些不好的书评，用户看到后不想购买，导致书籍的流转率下降；或者从用户角度
来说，对于某些用户，他们更喜欢自己去体会书中内容，不想带着别人对书的看法去阅读
；这样的话，书评功能或许是个累赘了。
从不同的角度看某些功能，也许会有不一样的理解。每一个功能都应该经过缜密的思考，
准确的用户调研，有价值的用户调研是清空自己的想法，接受用户的世界观。
一款好的产品是真正解决用户的问题，满足绝大多数用户合理的需求。
以上仅仅是我一些粗略的想法，还有很多不规范的地方，也想与大家一起交流一下关于漫
游鲸的这些问题：
为什么漫游鲸要将个人书架这个功能下架？ 面对直接竞品多抓鱼，为什么漫游鲸没有借
鉴多抓鱼的书评功能？ 大家更喜欢多抓鱼这款产品还是漫游鲸呢？
