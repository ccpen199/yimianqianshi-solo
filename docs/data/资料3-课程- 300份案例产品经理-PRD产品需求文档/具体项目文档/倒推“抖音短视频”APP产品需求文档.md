三、全局说明
3.1 功能权限
3.1.1 登陆/未登陆
登陆用户可以浏览所有界面，激活所有功能。 未登录用户只能以游客身份首页浏览推荐
视频，查看附近的视频，其他任何操作都需要登陆。
3.2 键盘交互说明
点击登陆页其他方式登陆，数字键盘从页面底部弹出。 点击其他输入框，字幕键从页面
底部弹出。
3.3 部分常用交互说明
3.3.1 打断后重新打开APP
3.3.2 点击空白区域或无网络状态
3.3.3 页面内交互
交互：首页推荐视频页面向上滑动，在1S内当前观看视频上滑动至顶部消失，新的推荐
视频从底部向上滑动出来播放。
交互：首页推荐视频页面向下滑动，在1S内当前观看视频向下滑动至底部消失，上一次
观看的视频从顶部向下滑动出来播放。


3.3.4 页面间交互
底部导航标：
交互：标签栏位于底部，点击相应的标签跳转到对应的页面，跳转时间为0.5S，
3.3.5 直播间交互
交互：
直播间右下角点击X返回抖音首页； 直播间右上角点击更多直播，则按照人气瀑布式展
示直播间； 抖音首页点击左上角直播间按钮进入随机（按照人气推荐）推荐的直播间。

3.4 页面异常

无网络状态下抖音首页提示
提示位置：首页页面中央
交互：
点击刷新，刷新页面； 左右滑动和上下滑动都触发刷新，依旧无网状态还是提示网络错
误。

无网络状态下直播间提示
交互：
没有网络状态下直播间屏幕变黑，页面中央提示“加载中”，直到有网状态下才能加载出
直播内容； 直播间处于“加载中”时候，点击更多直播间、关注等都无反应。

五、部分功能详细说明
5.1 注册/登陆页
手机号短信验证登陆：


微信登陆：


我的看法：
抖音作为一款创意短视频社交类APP，产品核心在于用户浏览视频观看别人分享的快乐，
用户在未登陆的情况可以观看推荐的优质内容，给用户留下良好的第一印象。 对于有登
陆需求的人来说，可以手机验证码登陆、一键登录、第三方登录（微信、QQ号、今日头
条）。一键登录无需注册成本，简化登陆流程引导用户更多关注社区内容跟互动。考虑到
其他用户场景，第三方授权登陆，减少用户填写个人资料的操作步骤，直接获取第三方个
人信息。
5.2 观看视频功能


我的看法：
抖音是一款主打UGC的产品，主页承载核心内容，观看推荐视频，发布作品。首页每次
自动播放一个推荐视频，降低用户操作成本，使页面更加简约，给人一种舒适放松的感觉
。 视频主要是幽默搞笑为主，相比较文字图片内容更加丰富，15~60S的短视频，以及评
论详情页，让用户体验成本不会那么重，瞬间让用户捧腹大笑，无意增加用户好感，提升
黏性。点赞、分享、收藏、评论等功能还可以满足用户互动需求。


5.3 搜索功能
我的看法：
搜索是一个内容社区类APP里不可缺少的内容，满足了用户精确寻找自己想看的内容。 
对于大多数用户而言，社会上热门事件、舆论等信息都是“被迫”接收信息，但这又是生
活中乐趣的一部分，搜索里的热搜榜等功能可以满足需求，比如：疫情话题、延迟高考等
。

六、非功能性需求
6.1 性能需求
响应速度：app启动时间不超过3秒，视频自动0.5秒内自动播放，只有无网络状态下不
播放。
缓存方式：直播间主播数据（头像、个人信息等）、我的个人信息（名字、抖音号等）、
用户未发送的作品选择草稿后选择本地缓存
首页内容采用懒加载模式，保障内容加载同时节约用户流量
6.2 运营需求
6.2.1 用户运营
账号加V：激励用户积极投稿，当用户达到某些条件时可获得身份标识“V”标，增加作
者的曝光、作者扶持等
6.2.2 活动运营
抖友一起抗肺炎：在疫情期间估计用户上传与疫情有关的正能量视频，并添加“抖友一起
抗肺炎”话题标签，投稿方向为“我为防疫出份力”、“快乐宅在家干点啥”、“防疫科
普小知识”、“防疫一线正能量”等等。引导社区内容，传播和谐正能量。
