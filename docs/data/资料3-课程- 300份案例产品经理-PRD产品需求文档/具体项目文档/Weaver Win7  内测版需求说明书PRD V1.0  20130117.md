	•	概述
本需求说明书为A产品视频通讯产品定义其PC平台的功能点。
	•	目的
该文档旨在明确A产品的相关产品规格，为开发人员、测试人员、交互设计人员提供需求说明，内容包括：产品需求，功能说明，数据统计需求等。
	•	适用范围
本PRD适用于PCwin7端的内测版版本产品。
	•	产品说明
此产品说明仅对涉及的相关功能点进行说明，对应UI文档版本的A产品_win7_203
	•	目标用户
具备亲情关系、爱情关系、亲密友情关系的用户群体。
	•	产品要解决的问题
为用户提供完整的视频通话功能，并帮助用户迅速熟悉A产品软件的使用
	•	竞品简要分析
竞品
我们的优势
我们的劣势
QQ
	•	基于熟人关系，联系人关系建立的门槛低。
	•	后期在手机通讯录关系的支撑下，比QQ关系更纯洁更活跃。
	•	引入家人关系上，有TV平台的核心优势。
	•	没有QQ泛化的关系网，收集视频通讯关系需要时间积累。
微信
	•	微信有运营商的利益压迫在，不能直接发起视频。我们基于熟人关系，直接发起的视频方式让用户体验更流畅。
	•	我们有TV平台做口碑铺垫。
	•	手机平台还未铺开，微信的触角已经几乎蔓延到每一个用户。
	•	如果微信与QQ打通视频通讯，并扫除用户体验的障碍。可能会构成较大威胁。
Skype
	•	本地化的产品氛围。在用户的到达上，具有先天优势。
	•	免费高清是杀手锏。
	•	Skype的用户关系圈较广，有陌生人交友的刺激。
	•	产品功能还未有Skype的丰满。
ooVoo
	•	竞品没有在中国落地，虽功能点和平台较我们全面，但受众面很小。对付国外竞品，我们有了解中国广大用户基层需求的优势。
	•	产品功能点上还没有ooVoo完善。

	•	上市时间：
此版本对应内测版本，与1月13日发布时间同步
	•	预装机型：暂无
	•	发行渠道：暂无
	•	产品预期目标
为用户提供完整的下载安装和卸载
解决与未注册用户的沟通和延时沟通，建立邀请渠道
导入旧关系、建立新关系。
打通建立用户关系渠道。
建立用户关系扩散机制。
用户关系的自动沉淀。
允许用户的个性化设置

帐户多平台登录策略
帐户多平台登录策略
1、多个设备同时在线，当有任何一个设备进入拨打状态时则该帐号已经进入拨打状态，其它设备无法进行拨打（提示当前帐号正在拨打中….），无法进行拨打只可以查找，但是无法拨通并进行提示。
2、多个设备同时在线，当有呼入电话请求时，所有设备都会弹出通话邀请窗口（TV端是否弹出需要参考“TV免打扰设置”策略），任何一个设备选择了视频接听/语音接听，其它设备的通话邀请窗口自动消失，此时该帐号进入拨打状态，参见规则1。
3、帐号可以同时在多个设备上登录（目前限定支持6个设备，并支持可扩展）
4、某一个设备上只支持登录一个帐号，同一个帐号可以同时在多个同种设备上登录（例如6个手机、6个PC登录同一个帐号）
5、任何一个设备上的帐号成功登录，则认为该帐号登录，全部帐号都登出，视为离线。
