# 《星云》大数据平台商业需求文档BRD（附模版）

BRD（商业需求文档）指的是基于商业目标或价值所描述的产品需求内容文档（报告），
其核心的用途就是用于产品在投入研发之前，是由企业高层作为决策评估的重要依据。
对于大数据在商业上的用途，就是找出更符合用户口味的产品以及相应服务，并且了解用
户的习性和兴趣爱好，这就是现大数据的核心价值理念。
当然对于每个人来说，心中的理念应该都是不同的，毕竟一千个人眼里有一千个哈姆雷特
。
笔者今天带来一份商业需求文档——BRD，对于BRD的内容需要做到简洁、易懂、一目
了然。因为当你要打造一款全新的产品时，这需要很多资源，你需要得到企业内的领导、
同事的支持。
对于一个企业来说“生存”是最重要的，我们需要用BRD去阐述为什么做该项目，阐述
怎么去解决项目的收益、成本风险及对策。
一、目录
针对目录要给予演讲对象一种一目了然的感觉，要知道，该PPT的汇报对象一般是你的
直属上级、CFO、COO、CTO、CEO，这几位中的一个或几位都是你所面对的阐述对象
，你需要让对方理解。


充分利用用户特点、用户行为、用户需求、用户场景进行真实性、针对性的描述整体概括
。
二、方案背景
方案的背景需要讲明以下几点：
市场的环境分析； 我们要怎么做； 需要解决什么样的问题； 利用我们的优势，得出可行
性的结论。
根据实际产品方案将以上几点阐述出来，例：
进入2012年，大数据（big data）一词越来越多地被提及，人们用它来描述和定义信息
爆炸时代产生的海量数据，并命名与之相关的技术发展与创新。它已经上过《纽约时报》
《华尔街日报》的专栏封面，进入美国白宫官网的新闻，现身在国内一些互联网主题的讲
座沙龙中，甚至被嗅觉灵敏的国金证券、国泰君安、银河证券等写进了投资推荐报告。
[1]  数据正在迅速膨胀并变大，它决定着企业的未来发展，虽然很多企业可能并没有意识
到数据爆炸性增长带来问题的隐患，但是随着时间的推移，人们将越来越多的意识到数据
对企业的重要性。正如《纽约时报》2012年2月的一篇专栏中所称，“大数据”时代已
经降临，在商业、经济及其他领域中，决策将日益基于数据和分析而作出，而并非基于经
验和直觉。
三、产品规划
在产品的规划这里我们需要明确产品的定性、产品的结构、产品的路线，当我们来到这一
步的时候谨记一步一个脚印，可以粗大的描绘不要过分追求细节。对于产品的规划核心要
思考怎么去做的问题，包括：
产品核心功能点； 产品架构； 产品阶段性规划； 产品未来路线图； 产品商业路线图；
我们需要用一句话或几个字来概括产品的核心，用一张思维导图展示产品的架构。主要是
将产品分为几个阶段来解决，做到有层次、有节奏、有规律，对于主要功能的规划要做到
：产品的核心功能明确、大致有哪些功能点、规划好产品发展路线。

四、盈利模式
互联网的产品有常见的九大商业模式：结合发展、垂直发展、以销定采、线上线下、开展
合作、合作创新、货币市场、媒介资源、个人合作、联合用户。曾经几时看到一篇文章内
阐述的一个结论，在这里引用一番：
商业模式错综复杂，而对于不同产品在不同的环境有不同的运营方式产生不一样的盈利模
式。但有三条结论是始终不变的：
用户价值大于获客成本，企业盈利； 降低获取用户的成本，提高用户LTV，运转速度加
快； 用户量达到某个点，用户价值倍增，来到爆发点产生集聚效应。
而产品的盈利模式，这是企业最关心或者说最需要了解的事项，可以理解为直接的收入，
当然，也可以是有利于企业长期发展的客观因素，比如：流量、资源、数据、模式；
五、收益与成本评估
收益预估包括收入、用户、品牌、降低成本、投入/产出比等，你需要将未来的收益与成
本进行一个初步的评估，当然，不用特别精确但是一定要心里有个大概，然后阐述时不要
将数据描述的特别理想，要知道市场对所有人都是很现实的。关于产品定价策略：
定价高策略：
短期内可获得高利润，掌握降价空间； 不利于打开市场低定价。
定价低策略：
可迅速打开产品销路，开启市场； 投资回收期较长，价格变动余地小，局限性大。
1. 成本
产品推广成本； 云平台； 人力成本； 推广成本； 基础设备成本。
2. 收益
会员服务费； 个性化服务； 广告业务； 商业合作。
针对风险我们需要对产品的各类风险有个大体了解，具体风险种类如下：

市场风险； 经济风险； 行业风险； 公司风险； 技术风险； 资金风险； 政策风险； 客户
风险； ……
不需要全部阐述，挑出几个比较重要的，针对性的去选择对策，根据不同的风险要有不同
的对策，一定要事先对风险这块有一定的认识，否则到Q&A环节你会很被动。


产品定价策略是战略产品定价时需要考虑三个主要因素：成本、需求和竞争产品。定价基
本上分为三种：1.成本导向定价法;2.需求导向定价法;3.竞争导向定价法。


总结
PPT做的比较简单，其实核心点都描述在内容中，重点在乎你要如何去阐述这些内容，
BRD只是一个能帮助你更好的表达商业规划的文档。所以要注重自身的表达能力，BRD
的大概内容板块如下：
方案背景（项目形成价值）； 方案价值（项目商业价值）； 产品规划； 盈利模式； 收益
与成本估算； 风险评估与对策。
