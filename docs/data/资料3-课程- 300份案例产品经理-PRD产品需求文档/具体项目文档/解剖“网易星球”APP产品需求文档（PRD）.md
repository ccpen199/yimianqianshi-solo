1.3 背景
2008年，中本聪发表了比特币的白皮书，描述了一种基于分布式账本的区块链架构的新
型货币体系。随着比特币的底层技术架构的扩散，各种加密货币开始发展，在全球范围内
吸引了越来越多人群的关注。
经过多年的发展，在2013年 Vitalik Buterin 提出了以太坊，它是以智能合约为功能核⼼
的的底层区块链系统。以太坊让开发者能够方便地在平台上开发智能合约和去中心化应用
，从而引领了去中心化应用的爆发。
现在，区块链开始和各行各业结合，市场上需要商业级的基于区块链的应用来为用户提供
服务。在这样的背景下，为了让用户分享个人信息价值和参与区块链的生态发展，网易星
球应运而生。
1.4 产品介绍
网易星球是网易推出的一款基于区块链生态的价值共享平台，在星球APP上，用户通过
浏览、学习、社交、购物等活动获得原力，从而获得每日按原力比例分发的黑钻奖励，星
球会帮助挖掘用户使用时产生的数据资产的价值，进而实现个人的数据资产的商业化，让
个人参与自身数据的价值分享。

1.6 名词说明
区块链：
区块链是一种按照时间顺序将数据区块以顺序相连的方式组合成的一种链式数据结构，并
以密码学方式保证的不可篡改和不可伪造的分布式账本。
原力：
原力是用户获取黑钻的影响因子，在同一时段内，原力越高，获取的黑钻越多。
原力分为永生原力和魔法原力两种类型。永生原力一般数值较大，但有固定生效期，仅在
生效期内可参与黑钻分配，超过期限将会自动失效。可以通过在星球上进行浏览、学习、
社交等活动，获得原力，由于目前星球基地正在前期搭建中，未来将支持更多的原力获取
方式。
黑钻：
黑钻是依托于区块链技术，基于个人星球活动产生的奖励，可以用于星球上的消费与兑换
等。 除日常根据原力大小生长黑钻之外，原力大于35的用户有机会获取额外的黑钻大奖
，称之为幸运钻。48小时不领取黑钻将暂停生长。黑钻总量有限，且每2年产出量减少
一半，随着时间的推移获取难度越来越大，前期参与更有优势。
数据护照：


数据护照是连接电商、游戏、娱乐、金融、健康五大数据系统和个人数据价值的枢纽。通
过在区块链上建立的数据护照来进行个人数据所有权的确权和隐私保护。
黑钻钱包：
黑钻钱包就像银行卡一样有账号和密码，黑钻钱包的区块链公钥就类似于银行卡号，黑钻
钱包的区块链私钥类似于银行卡密码。在区块链上记录着钱包地址对应的黑钻数量。

三、全局说明
3.1 功能权限
已登录：可以使用APP内所有功能； 未登录：只能浏览“首页”、“原力任务”、“我
的”这些一级页面，在“首页”中使用任何功能都会弹出对话框提示用户未登录，在“原
力任务”、“我的”页面中使用任何功能都会直接跳转到登录注册页面。
3.2 键盘说明
输入手机号码、验证码、密码等纯数字信息时从页面底部弹出数字键盘； 输入其他内容
时从页面底部弹出字母键盘。


3.3 网络异常
当网络异常导致页面加载失败时，页面中提示用户网络出错刷新重试，点击“刷新”按钮
会重新加载页面。

四、主要业务逻辑流程
4.1 登录注册
登录注册流程如图8所示，网易星球在登录注册页面没有区分登录和注册，通过输入手机
号码和验证码后提交，然后拿到验证结果后得到是否已经注册，如果已经注册那么登录成
功，如果未注册继续走注册流程，注册成功后自动登录成功。

图8 登录注册流程图
4.2 业务主流程
网易星球业务主流程围绕原力、黑钻、数据展开，用户需要完成各种通过原力来激励的任
务来增加原力值，系统会根据用户当前原力值生产出用户的黑钻，用户领取系统生产出的
黑钻来获得收益，通过享受黑钻的权益和参加黑钻的活动来使用黑钻。
当用户做原力任务时会产生行为数据，网易星球会将经过用户授权的数据上链，进而让用
户的数据得到公平公正的价值交换，实现个人信息数据的价值商业化。

五、产品详细功能说明
5.1 启动页
图10 启动页交互图
页面逻辑描述：


空白页中简洁的显示产品Logo、产品名称、产品口号；广告页在空白页的内容基础上增
加广告内容和跳过广告功能；广告详情页是跳转到的具体的广告内容页面。
页面交互说明：
打开APP会首先进入到“A空白页”，然后经过2秒根据网络情况和广告数据加载情况
判断是否需要展示广告，如果不需要展示广告会进入“D首页”，如果需要展示广告跳转
到“B广告页”； 在“B广告页”点击广告会进入“C广告详情页”，点击“跳过”或者
经过5秒不操作自动跳转到“D首页”。
思考：
启动页完成没有跳转到登录注册页，而是跳转到首页，首页展示了星球的获取黑钻、多种
原力任务、黑钻活动等核心功能，让用户先了解星球的主要功能，然后再引导感兴趣的用
户注册，比直接跳转到用户注册页面带来更高的转化率。
但是，星球目前应该还处于早期阶段，最近刚刚公测，整个体系还需要进一步完善，在这
样的情况下尝试在APP启动时插入广告进行商业化变现，会对用户的使用体验和产品的
传播口碑有一定的影响。

5.2 登录注册
图11 登录注册交互图
页面逻辑描述：
在“登录注册”时通过输入手机号码和验证码来进行登录和注册；“获取验证码”时会先
判断手机号是否有效，无效弹出Toast提示；如果是用户首次使用，会进入注册流程，输
入昵称、邀请码（选填）和实名信息后完成注册；如果用户已经注册过那么完成登录直接
进入首页；在帮助页面提供了常见问题。
页面交互说明：


（1）在“A登录注册”页面，点击“帮助”会跳转到“常见问题”页面；点击输入手机
号、验证码文本框，数字键盘从底部弹出；输入时可点击右侧图标删除输入的信息；点击
“获取验证码”时手机号码无效时弹出Toast提示“ 网易星球：请输入正确的手机号”
，手机号码有效时按钮置灰，按钮内容变为“重发（60秒）”从60秒倒数，60秒后按
钮内容恢复为“获取验证码”，按钮颜色也恢复成原始的色度；点击“立即开启”后在页
面中间转菊花，等待返回结果，如果成功那么进入下一步，如果失败弹出Toast提示“网
易星球：验证码不正确”。
（2）在“B设置昵称”页面，点击输入邀请码和设置昵称时会弹出拼音键盘，邀请码是
选填，昵称是必填，然后点击“下一步”进入到“C实名信息”页面输入姓名和身份证号
，输入姓名和身份证号时弹出拼音键盘，点击“确定”如果不正确会给用户Toast弹框提
示，如果正确完成注册流程，然后跳转到“我的”页面。
思考：
用户除了浏览星球主页面之外进行任何操作都会引导到登录注册页面，因为星球的进一步
操作都需要有用户信息，比如：领取每日生成的黑钻、通过完成原力任务获得原力、查看
账户信息等，这样的设计在一定程度上平衡了用户体验和拉新用户。
在登录注册时没有明显区分登录和注册，通过将“复杂留给自己，将简单留给用户”来减
少登录流程和提升用户体验。目前星球提供了手机短信验证码登录的方式，如果后期应用
的体量和用户的规模越来越大时可以考虑提供邮箱、微信等其他方式登录，而不需要通过
获取短信验证码登录，也能减少运营成本。

5.3 首页
图12 首页主要信息交互图
页面逻辑描述：
首页中包括展示黑钻数量和原力值、公告信息、每日生产的黑钻、活动信息；活动信息里
面有推荐的原力任务、黑钻活动、黑钻权益等。在黑钻记录中会看到黑钻的数量、简介和
收支记录；在原力记录中有原力值、简介和记录。
页面交互说明：


在“首页未登录”中，点击首页内容会弹出对话框提示用户登录；如果已经登录，那么会
有每日生产的黑钻供用户点击领取，点击领取时页面中黑钻数量同步变化；点击黑钻数量
区域跳转到“D黑钻记录”页面，点击原力值区域跳转到“E原力记录”页面，点击公告
区域跳转到“F星球公告”页面。
图13 首页活动信息交互图
页面逻辑描述：
首页中的活动信息里通过简洁的介绍每种原力任务、黑钻活动、黑钻权益组成列表，这些
信息每次打开页面都从后台动态获取更新，用户点击后进入对应的任务和活动页面进行下
一步的操作。
页面交互说明：
在“A首页”中点击原力生产区域的单个活动列表项（例如：黑钻挖宝、双节嘉年华等）
进入对应的活动页面；在“A首页”中底部多个活动列表项横向列出，通过左右滑动展示


各列表项，点击活动列表项跳转到对应的“B购物”、“C游戏”、“D小说”、“E资
讯”、“F漫画”、“G竞猜”、“H挖宝”等页面
思考：
“首页”模块从星球发布到目前的版本一直以让用户领取每日生产的黑钻为核心，在这基
础上逐步增加了活动和任务相关的玩法，吸引用户参与到原力任务中生产数据、参与到黑
钻活动和黑钻权益中享受福利。
48小时不领取，黑钻将会停止生长，这样的规则一定程度上会提升用户的活跃度。用户
愿意参加原力任务和黑钻活动是建立在认可黑钻价值的基础上，如果黑钻没有价值，用户
就会失去玩星球的动力，所以让用户继续提高对黑钻价值的认可是星球需要继续探索的地
方。
目前星球在单向让用户获取黑钻方面的产品设计较为丰富，在反向让黑钻消耗和流通方面
可以继续深耕，例如：推出黑钻红包、黑钻兑换原力任务中的商品和服务、黑钻商城等功
能。

5.4 原力任务
图14 原力任务签到交互图
页面逻辑描述：
在原力任务页面的发现原力区域中显示原力值和签到领取“魔法原力”，通过签到功能能
够每天增加原力值。
页面交互说明：
点击“签到”弹出对话框和遮罩遮住其他信息；点击对话框中的“不用了”或者关闭按钮
，对话框和遮罩消失，页面恢复原样，原力值更新；点击“去探索”进入宇宙探索游戏页
面。


图15 原力任务原力分类交互图
页面逻辑描述：
在原力分类区域分成了购物、游戏、娱乐、来钱、健康五类，购物类别包含淘宝、考拉、
严选，游戏类别接入了网游、小游戏，娱乐类别里有小说、漫画、新闻、音乐，来钱作为

页面交互说明：
点击“购物”、“游戏”、“来钱”分类图标会进入对应的分类页面；点击“娱乐”会进
入二级分类页面，点击二级分类页面中的“小说”、“漫画”、“新闻”、“网易云音乐
”会进入到对应的详情页面。


图16 原力任务其他交互图
页面逻辑描述：
在推荐原力区域，将原力分类中的需要推荐的原力任务提取出来放到列表中展示，此外还
增加了“邀请好友”和“关注微信公众号”的原力任务。
页面交互说明：
点击“推荐原力”中的原力任务跳转到对应的页面，点击“邀请好友”进入邀请码页面，
点击“关注微信公众号”，进入页面后输入获取到的验证码，验证正确显示任务完成对应
信息。
思考：
原力任务页面通过奖励原力值来激励用户进行购物、浏览、游戏、邀请、关注等行为，能
够产生星球生态需要的数据，这样为进一步价值交换提供了基础，而且通过用户的做原力
任务既可以为第三方带来流量从而为平台产生利润，也能为平台带来新粉丝、新用户。
但是，原力任务过多过杂时会让用户产生广告的感觉，建议随着星球的发展在构建原力任
务时可以考虑根据用户的行为习惯进行个性化的任务推荐。


5.5 我的
图17 我的交互图
页面逻辑描述：
该模块主要是罗列用户在星球的数据和信息，在“我的”页面包括用户昵称、居民类型、
我的资产、我的邀请码、星球基地、设置，其中我的资产包括我的钱包、数据护照、我的
原力值。
在我的钱包详情页会有我的资产、管理钱包、区块链地址、我的私钥以及交易记录。在数
据护照页面将用户的基本信息、数据分析、使用情况、权益整合出来显示。在原力记录页
面包括原力值、简介、记录。在邀请码页面显示的是该用户的邀请码和邀请信息。在星球
基地页面中含有口号、介绍、黑钻总产量、黑钻昨日产量、个人价值挖掘、建设规划等信
息。


页面交互说明：
在“A 我的”页面点击“我的钱包”、“数据护照”、“我的原力值”、“我的邀请码”
、“星球基地”、“设置”进入到对应的页面。 在“我的钱包”页面点击“管理钱包”
从底部弹出对话框进行管理，点击“区块链地址”在页面中间弹出对话框显示地址信息，
点击“我的私钥”从底部弹出包含数据键盘和输入密码的控件，点击“交易记录”在页面
中间弹出对话框显示详细的交易信息。 在“G账户设置”页面点击“退出登录”弹出对
话框二次确认退出账户。
思考：
在该模块将用户在星球的账户信息、资产信息、数据护照、星球基地信息呈现给用户，让
用户逐步了解和管理个人在星球的资产和数据，从而为未来实现去中心化数据信息交换奠
定了基础。
六、总结
网易星球作为网易布局区块链的重要产品，目前已经包含较多的操作页面和复杂的功能信
息。根据当前的星球APP版本，本文解剖了其主要功能的需求文档，分析了主要的功能
流程、业务逻辑、页面逻辑和交互说明，阐述了个人的思考和建议。由于时间关系，对于
星球APP的消息推送、下载更新、原力任务的详细页面没有来得及进行详细分析。

不足之处恳请大家指出来共同交流。
