# 宠爱APP需求文档

编辑导语：如今养宠物的人越来越多，宠物市场也愈发完整；现在都讲究科学喂养宠物，
大家对宠物的饮食安全和日常护理都非常重视，有一款专门负责宠物的APP是非常有必
要且需要的；本文作者撰写了一个宠爱APP的需求文档，我们一起来看一下。

1）产品定位
宠爱是一款专注服务于宠物和爱宠人士的综合服务类的社交 APP。
这里有广阔的平台给铲屎官记录分享与爱宠的生活日常，提供海量的专业养宠知识，为爱
宠的健康保驾护航；专属的宠物档案记录爱宠的数据指标，为饲养员提供事件提醒及贴心
健康提示；同时宠爱还会不定期组织各类宠物活动，线上线下一起嗨！

3. 需求整理
1）需求分析
2019 年中国宠物（犬猫）消费市场突破 2000 亿的关卡，整体消费规模达到 2024 亿元
，比 2018 年增长 18.5%。
其中，宠物犬消费市场规模为 1244 亿元，比去年增长 17.8%；宠物猫消费市场规模为 
780 亿元，增幅达 19.6%。（来源：《2019 年中国宠物行业白皮书》）


《2019 中国宠物消费趋势报告》报告显示，近年来中国宠物市场规模正在持续扩大，未
来 3-5 年内，宠物行业将继续维持 14%以上的增长。
养宠人群属性方面，以女性为主（占比 88.4%），90后是主要养宠群体，未来 95 后将
承担消费主力。
其中养宠物主人中大专以上学历占比约 84.8%，高消费人群占比约 20%；总体来说，宠
物主人在逐渐大众化的同时，也有趋向年轻化、高收入、高学历的特征。（来源：《
2019 年中国宠物行业白皮书》）。


由此可见，未来随着人们的收入水平提高、消费结构改善、教育水平提升，国内宠物行业
将持续稳定发展，需求也将会越来越大。
面对不同人群的不同需求，需要一个平台将这些宠物主人聚集起来，用以记录分享宠物生
活、购买宠物用品、交流养宠经验；同时也为由于各种原因暂时还未养宠物、或者准备养
宠物的用户提供“云”吸猫撸狗、咨询平台。

三、全局说明
1. 功能权限
未登录/登录：
已登录状态可执行所有操作； 未登录状态下，仅供用户浏览社区、问答、活动模块内容
信息，用户无法进行其他APP操作；当进行除浏览外的其他操作后将直接跳转至登录页
面。
2. 登录/注册页面


1）页面名称：登录/注册页面。
2）页面入口：未登录状态下，不能点赞、评论、分享、回答、私信；不能发布动态及提
问；不能查看我的活动、个人信息、宠物档案、收藏夹、备忘录、草稿箱；当进行以上操
作时，自动跳转至登录界面。
3）页面功能：三种方式登录——账号/手机号+密码登录、手机号+验证码登录、第三方
授权登录；注册账号。
4）页面逻辑内容及交互详细说明：
a. 账号/手机号+密码登录


初始界面下，呈现账号/手机号码输入栏、密码输入栏、忘记密码按钮、登录按钮、注册
按钮。 账号/手机号码输入栏默认+86，账号/手机号码和密码输入框均没有字符类别限制
，字数限制为6~16个字符。 密码输入框输入密码后，可点击右侧可视图标，切换密码
显示状态。 输入完成后，点击登录，验证正确完成登录，跳转至进入登录前界面 点击“
忘记密码”按钮跳转到找回密码页面，输入手机号获取验证码，并输入新密码进行密码重
置。
b. 第三方授权登录
初始界面下，在登录按钮下方，呈现”第三方账号登录“按钮。 点击按钮，提供微信、
QQ平台按钮，选择后点击，进入此社交平台授权登录界面。 授权完成后，完成登录，跳
转至进入登录前页面。
c. 手机号+验证码登录
初始页面下，在密码输入栏左下方，呈现”验证码登录“按钮。 点击按钮，进入验证码
登录页面。 在验证码页面输入完成后，点击登录，验证正确完成登录，跳转至进入登录
前界面。
d. 注册账号
点击“注册账号”，进入“账号注册”页面，通过手机号验证码注册账号。 当前界面下
，呈现手机号码输入栏、验证码获取按钮、验证码输入栏、密码输入栏、注册按钮。 手
机号码默认+86，输入限制11位数字。当输入非数字内容时，输入界面不显示任何内容
；当输入大于11位数字时，超过部分不显示。 密码输入框无字符类别限制，指数限制为
6~16个字符。
3. 键盘说明
点击手机号码输入框时弹出数字键盘。
点击其他输入框弹出字母键盘在页面内。
4. 部分常用交互说明
1）页面间交互

点击底部标签导航内“社区”、“问答”、“发布”、“活动”、“我的”五个标签，切
换至对应页面。 在页面内下拉时，刷新页面；上拉时，加载内容。 除底部导航栏的五个
一级页面，其余页面在返回上一页面时，页面向右滑出。 从“搜索”页面返回上一页面
时，页面从底端向上滑出。 不同页面之间，无手势切换。
2）打断后重新打开 APP
当用户返回桌面或者临时接入电话后重新打开APP，将直接进入退出前的页面。 当APP
出现异常关闭、闪退、崩溃情况时，用户重新打开将进入APP启动页。 当APP页面临时
出现短信或者其他信息通知时，APP页面无影响。
3）其余通用形式
如APP当前页面有软键盘时，点击页面内空白区域，软键盘向下滑动隐藏。 无网络情况
时，每次打开APP时页面均会弹出Toast，内容为“网络出错了，请检查网络后重试”，
点击其他区域弹窗消失；用户可以在无网络情况下浏览APP，但页面内无内容加载；当用
户进行除页面切换之外的操作时，提示弹窗将再次出现。

六、页面功能详细说明
1. 社区页面
1）页面名称：社区页面。
2）页面入口：启动宠爱APP直接进入“社区”，或在其它页面时点击底部tab栏的“社
区”按钮进入。
3）页面构成：顶部工具栏、动态内容展示、底部tab栏。
4）页面逻辑内容及交互详细说明：
a. 顶部工具栏
搜索框：未点击时搜索框出现推荐搜索内容字样；点击进入搜索页面，推荐搜索消失，页
面展示搜索内容一级分类（今日热点、话题、宠物、问答、活动、分享）和二级分类；选
择一级分类时，二级分类部分对应变换；仅选定的一级分类按钮变亮，其余一级分类按钮


为灰暗；选择二级分类时，点击二级分类图标，页面跳转至对应页面。 消息详情：点击
图标进入消息页，展示系统信息（服务通知、社区、问答、活动通知）和聊天信息（聊天
人头像、昵称、最新聊天内容预览）；点击不同消息栏，页面跳转至对应页面。
b. 动态内容展示
展示用户头像、昵称以及当条动态的预览(话题、文字、视频、图片)和对应的评论数、点
赞数、收藏数；点击动态的预览内容(话题、文字、视频、图片)进入动态详情，点击评论
进入评论详情。 点击动态内容展示区进入动态详情，展示用户基础资料（头像、昵称、
签名）、当条动态(文字、视频、图片、话题、发布时间、定位)、动态对应的评论数、点
赞数、收藏数，以及热门评论（评论者头像昵称、评论内容、评论点赞数）。长按图片可
选择保存或举报，可点赞评论收藏分享。 进入动态详情页，可选择点击用户头像右侧的
关注按钮，关注前按钮为蓝色，关注后按钮呈灰暗。点击页面底右上角分享按钮，可选择
QQ、QQ空间、微信、朋友圈、复制链接五种方式，点击后将跳转至对应页面。
c. 底部tab栏
APP的主菜单，为“社区”、“问答”、”发布“、”活动“、”我的“五个主要页面的
入口，点击不同按钮进入对应页面，点击中间十字按钮，进入”发布“页面。 点击十字
按钮进入发布选择页，页面可选择“文字”、“图片”、“视频”、“提问”四种内容发
布，点击后跳转至对应内容编辑页。 除发布页外，其他所在页面按钮变亮，其余页面按
钮为灰暗。

2. 回答详情页面
1）页面名称：回答详情页面。


2）页面入口：点击进入底部tab栏的“问答”按钮后，再点击问答推荐列表中的某个问
答进入。
3）页面构成：问题内容展示、操作区、回答内容展示、评论内容展示、评价区。
4）页面逻辑内容及交互详细说明：
a. 问题内容展示
展示提问内容（文字、图片）、问题分类以及该问题的回答数。 点击图片可放大图片查
看；点击查看问题全部答案，页面将跳转至问答详情页。
b. 操作区
该区由”邀请回答“和”我要回答“两个按钮构成。 点击“邀请回答”按钮，页面跳转
至邀请页面。邀请方式有三种：顶部搜索框搜索好友、中部互相关注栏选择好友（该栏最
多显示最近联系的八位好友）、底部邀请更多栏分享QQ好友或微信好友；点击搜索好友
页面跳转至搜索结果页面；互相关注区点击邀请按钮后，按钮变灰暗且内容变为“已邀请
”；点击分享QQ好友或微信好友按钮，页面跳转至对应页面。 点击”我要回答“按钮
，页面跳转至内容编辑页；页面支持文字输入外，还支持插入图片、视频。点击发表页面
将跳转至新发表回答详情页面。
c. 回答内容展示
展示回答者信息（头像、昵称、签名）、回答内容及回答发布时间。
d. 评论内容展示
展示评论者信息（头像、昵称）、评论内容、评论点赞数。 点击更多按钮，页面将跳转
至评论详情页。
e. 评价区
喜欢、收藏按钮：展示喜欢数及收藏数，点击喜欢、收藏图标后，图标变为红色，数量对
应+1；再次点击图标可取消操作，图标、数量恢复操作前状态。 输入框：未点击时输入

框出现“说点什么吧”字样；点击搜索框，页面底部会弹出字母键盘，页面内容随着键盘
弹出上移。
3. 活动页面
1）页面名称：活动页面。
2）页面入口：点击底部tab栏“活动”按钮进入。
3）页面构成：我的活动、活动分类、活动信息展示。
4）页面逻辑内容及交互详细说明：
a. 我的活动
点击“我的活动”，页面跳转至历史报名活动页面；页面展示历史报名活动名称、缩略图
及活动状态（挑战成功、挑战失败、已结束、进行中）。
b. 活动分类
活动分类为“报名进行”、“即将开启”和“往期活动”三类，默认进入活动页面显示“
报名进行”内容，点击不同按钮，切换对应活动信息展示。
c. 活动信息展示


展示活动名称、缩略图、报名人数、报名时间信息，活动信息栏内容按照时间从新到旧排
序。 点击活动信息栏进入活动详情页面；页面展示活动详情信息及报名按钮，向下滑动
可查看更多活动内容；点击“我要报名”按钮后页面弹出Toast，内容为“恭喜您，报名
成功！”，3S后将自动消失；同时按钮内容变为“已报名“；再次点击按钮可取消报名
。
4. 宠物档案页面


1）页面名称：宠物档案页面。
2）页面入口：点击进入底部tab栏的“我的”按钮后，再点击宠物档案列表中的某个宠
物名片进入。
3）页面构成：宠物基础信息展示、功能区。
4）页面逻辑内容及交互详细说明：
a. 宠物基础信息展示
展示宠物图片、昵称、性别、年龄、性别、身体情况、生日及犬证编号信息。 点击图片
可放大图片查看，长按可选择更换照片、保存图片。
b. 功能区
健康管理：进入宠物档案页面，功能区默认显示健康管理内容；健康管理由日历及“日常
事项”、“健康管理”、历史记录“三个按钮构成；点击”日常事项“进入内容编辑页；
点击”日历“、”健康管理“、”历史记录“进入对应内容查看页。 成长日记：点击“
成长日记”按钮，功能区内容切换成对应内容，可向下滑动查看更多日记记录；点击日记
内容，页面将跳转至对应详细页面。
七、总结
“中国宠物产业虽然起步晚，但是发展快，我们甚至用10年，走完发达国家的30年”
（摘抄自《2019年中国宠物行业白皮书》）。
随着经济的高速发展，以及对养宠的态度改变，未来一段时间内宠物的市场将会愈发壮大
，且前景不可估量。
目前养宠人士整体趋向年轻化，尤其在如今这个社交至上的社会，宠物也成为了现代年轻
人社交的一部分。
宠爱APP作为一个宠物社交平台，能够帮助解决用户宠物社交的需求，并且还能够通过
各种活动将这种社交蔓延至线下。

除此之外，宠爱APP还有问答与宠物档案的模块，能够为养宠人士喂养宠物的全阶段提
供服务；由此宠爱APP也与市面上现有的宠物社交APP存在一定差异。
后期宠爱APP可以从社交强度和内容质量着手改进，后期迭代也将对这两方面进行加强
与提高。
