作为一款主打UGC的产品，皮皮虾凭借其缤纷多样、趣味十足的内容让用户“上瘾”，
并赢得了不少用户的好评与持续性使用。那么这款产品背后的产品架构与逻辑又是怎样呢
？他们是如何支撑起这款有趣的产品的呢？笔者将为大家仔细分析。

五、部分功能详细说明
5.1 注册/登录页
手机号验证码登录


微信登录：


手机号密码登录：


我的看法：
皮皮虾作为一款搞笑内容社区类APP，产品的核心价值在于分享快乐，用户在未登录的情
况下即可直接浏览社区内容获得快乐，看完即走，凭借优质的内容给用户留下良好的第一
印象。
而对于有登录需求的用户来说，登录分为验证码登录，第三方（微信、qq）登录和密码
登录。
皮皮虾把醒目的一键登录作为登录入口，无需注册，本机快速登录，简化登录流程以引导
用户更多关注于社区内容与互动。
考虑到其他的用户场景，第三方登录导入了第三方软件的个人信息，减少用户填写个人资
料的操作步骤；而密码登录也为商城购物的用户对一键登录的安全性担忧提供了一层保障
。
可以看到所有的登录方式最终都需要与手机绑定。手机号的获取无疑提高了用户画像与后
台数据的质量，在满足网络安全法规的同时也为产品迭代和后续的运营提供了更加真实可
靠的数据基础。
5.2 首页
（1）看帖功能


我的看法：
皮皮虾是一款主打UGC的产品，其主页作为核心模块承载了所有内容分发功能。帖子以
卡片为容器，使界面分布更简约、富有层次感的同时又具备了良好的可用性。
帖子内容以幽默搞笑的视频为主，相比于文字和图片，这种动态的内容直接在卡片上播放
会迅速吸引用户的注意力，配合幽默或视角独到的神评，瞬间让用户捧腹大笑，无意中增
加了用户的好感度，提升了用户粘性。点赞、关注、分享、评论、回复等功能亦满足了用
户的互动需求。
从心理学角度来看，15秒左右的短视频恰好符合了人的专注力最能集中的时长，如果时
间太长，注意力的聚焦程度则会大打折扣，而15秒的时间，刚好能够让人形成片刻的印
记，产生强烈的愿望，想要再看一遍，或者看下一条，加上大数据的精准推送同类别的内


容，强化了用户从感兴趣的类别的内容中获得的持久的，高强度的愉悦感，提升了用户粘
性和忠诚度。
从功能结构来看，帖子内容分为帖子主题和评论列表，优质的评论依据点赞数自上而下排
列，而评论列表又内含评论详情，塑造了楼中楼的内容呈现模式。
这种模式包含了大量信息的同时又对优质内容进行了筛选排序，提升了用户获取优质内容
的效率，当用户点击进前几条优质评论详情里又会发现关于这些评论更多的精彩内容，给
与用户一种不断挖到宝的感觉。
（2）搜索功能
我的看法：


搜索是一个在内容社区类APP里不可或缺的功能，满足了用户精准快速寻找到自己想看
内容的需求。
然而对于一个搞笑幽默类内容的消费者来说，TA来这个APP主要是寻找消遣，放松心情
的，并不会“带脑子”主动思考自己想要看什么，更多的行为是刷到什么看什么，不喜欢
的直接跳过。
因此只有在一些特殊场景中，譬如想看最近的热点话题“疫情“或者“囧妈”的用户才会
有选针对性的使用搜索功能。
为了满足用户想看热点的需求，热搜榜单被填充进了搜索页面，而对于那些首页里没有找
到感兴趣的帖子的用户，进入搜索功能看到“虾看视频榜”和“评论封神榜”也会不自觉
的点进去，看看每日播放量最高的视频和评论被点赞最多的帖子是否真的那么有意思。
（3）看电影/剧功能
我的看法：


看电影/看剧功能对于提升用户粘性做到了很好的辅助作用。
皮皮虾作为“头条系”产品，“成瘾性”也是它的特点之一。当产品只有单纯的看帖功能
时，用户假如并没有被帖子吸引很可能会离开产品，因此为了保证用户的活跃度，提高用
户花费在APP内的时间，为他们提供另一种完全不同的产品体验可以给用户带来新鲜感
，增加继续探索产品的时间。
因此为了激发用户的“瘾”，放映区功能的上线也预示着皮皮虾的战略方向从当初纯看帖
的搞笑幽默类内容社区进化成多元化“皮”娱乐的内容社区。
从页面结构上来看，电影页面由电影播放器、弹幕和评论组成，这样一来，用户在看电影
的同时参与底下的评论，或与其他观众在银幕上发弹幕互动，产品把一个个独立的观众连
接起来形成一个观影、讨论电影的团体，满足了观影和社交需求。
我的建议：
电影和电视剧的数量还是太少，种类覆盖不够全面，而且大多数都是几年十几年前的老电
影，老港片。希望能够多购买点版权，多上架一点最近几年的电影，丰富品类。 电影和
电视剧没有分类筛选功能，所以找电影只能不停的往下滑动页面，用户体验很差，就好像
所有电影都挤在一个大箱子里，要一个一个去找。因此希望增加筛选电影和电视剧的功能
，按时间、地区、种类、评分等类别分门别类，以便优化用户体验。

5.3 直播页


我的看法：
直播是19年才新上架的功能，经过我的观察，APP内没有用户开通直播的入口，主播只
有在获得了邀请才能开通直播，直播间的一些基本功能比如礼物系统已经建立起来了，但
观看人数以及主播数量上都不算大，可能暂时还处在测试阶段。
至于引入直播功能的原因，我认为：
①对部分用户的二次吸引：可能某些用户没有感兴的帖子或者影视，这时候很有可能直接
退出APP，直播功能提供了二次吸引的机会，强行将用户拉回来看看有没有吸引自己的直
播，提高了用户留存度。


②为创收做准备：目前皮皮虾的商业化手段主要就是广告植入和商城售卖周边产品，但作
为一个已经有500万DAU的产品，更多的变现手段意味着更多的盈利，因此通过直播的
打赏也能为公司带来更多的效益。
③转型多元化娱乐社区的功能点之一。至于直播的效果最终如何，还有待继续观察。
5.4 发帖页


我的看法：
优点：低门槛的视频编辑/拍摄门槛，并且增加趣味性丰富的配乐，特效功能
皮皮虾的视频编辑都有皮一下的功能，通过简单地给视频增加贴纸、特效、配音等娱乐元
素增强了趣味性，让没有视频制作经验的用户都能制作出高水准的视频，激发了用户的创
作需。
缺点：
①发链接功能对链接有特定格式的要求，用户不易理解，使用频次几乎为零。
②贴纸、特效和配乐资源不够丰富，限制用户创作的想象空间。笔者曾经想尝试发布一个
视频，但发现配乐只有5个类目，总计不超过100首歌曲可以选择；特效与贴纸融合在
一起，总计特效的数量只有34种。因此只能将视频下载下来通过其他短视频软件进行编
辑再上传至皮皮虾，使用体验很不好。
我的建议：希望能够优化发帖的流程，去掉发链接的功能，并且添加更多更有趣的特效和
配乐，方便用户创作更多更好玩的内容。

六、非功能性需求
6.1 性能需求
性能需求
响应速度：app启动时间不超过3秒，WiFi网络下视频0.5秒内自动播放，只有移动网
络时视频不自动播放。


缓存方式：已看的帖子内容（图片、视频），直播间主播数据（头像、个人信息等）、我
的消息页内容、用户未发送的帖子在选择保存草稿后的采用本地缓存。
首页内帖子图片采用懒加载方式，保障页面加载速度同时节约了用户流量。
6.2 运营需求
6.2.1 用户运营
神评鉴定师：鼓励用户通过发布优质评论以获得神评鉴定师称号。用户在获得一定数量的
赞或者写出2条神评后即可获得神评鉴定师称号，神评鉴定师将专享一键“送神评”权限
并获得专属勋章。此举可推动用户自发性的提高投稿和评论的质量，进一步打造优质内容
社区。 账号加V：激励用户积极投稿，当用户账号达到某些标准可获得身份标识“V”标
，并享有增加作品曝光，作者扶持及专属礼品等特权。 皮皮虾勋章墙：分为神评勋章和
宣传大使勋章，鼓励用户发掘优质内容、帮助推广，可获得对应勋章及其他相应奖励，彰
显个人身份。
6.2.2 活动运营
同城皮友群：加入同城皮友群，畅聊互助巨友情。鼓励同城用户进行产品外交流互动，打
造产品自身“皮文化”。 宅在家音乐直播周：活动时间内关注主播并观看主播唱歌。引
导用户使用直播间功能关注主播，看直播。 皮友一起抗肺炎：在疫情期间估计用户上传
与疫情有关的正能量视频，并添加“皮友一起抗肺炎”话题标签，投稿方向为“我为防疫
出份力”、“快乐宅在家干点啥”、“防疫科普小知识”、“防疫一线正能量”等等。引
导社区内容，传播和谐正能量。
6.2.3 内容运营
皮皮虾厨艺大赛：引导用户投稿，提高用户创作欲望和社区活跃度。主题为在家做/吃美
食，带上#皮皮虾厨艺大赛 标签，必须为原创稿件。 话题运营：通过话题来促使用户产生
共鸣和倾诉欲望。鼓励用户对某一话题进行相关创作，投稿时需添加该话题标签，展示作
品时会显示话题，提升用户活跃度的同时亦可增加话题的曝光量，引导更多的人关注该话
题。 放映厅推荐页上架《囧妈》、《荒岛余生》等新片，吸引用户点击观看。
