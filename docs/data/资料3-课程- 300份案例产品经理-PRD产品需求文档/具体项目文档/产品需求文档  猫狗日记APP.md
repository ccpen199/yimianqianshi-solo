# 产品需求文档  猫狗日记APP

本文主要对猫狗日记APP产品进行了产品需求分析，并展开了一份多维度且详实的产品
需求文档。该文档由产品说明、产品整体框架、产品使用流程和产品原型说明等几个部分
组成，并在最后总结了对猫狗日记APP的几点建议，与大家分享。

1.2 产品简介
1.2.1 产品定位
猫狗日记是一个专注于宠物的社交app，为养宠者提供经验交流、宠物日常分享和养宠问
答，为有意愿养宠的宠物爱好者提供了解学习养宠、“云领养”（云领养后可以查看自己
领养的宠物更多的视频、照片等等、以达到模拟养宠的目的）以拥有属于自己的“云宠物
”。用户可以透过平台了解养宠的相关知识和云领养自己喜欢的宠物，以及想了解的宠物
的一切。
1.2.2 竞品分析


在app store中搜索关键词：宠物、社交，出现的结果中超过1000评论的app有犬易、
汪汪萌主、犬易、OwOh、小狗在家。此四款app定位各有差异：犬易主要是社交+商城
的模式；汪汪萌主是以宠物社交和宠物为媒介的养宠者交友平台；OwOh也是社交+商城
的模式；小狗在家社交+宠物寄养的模式。
除了OwOh以外，另三款app都是仅有宠物狗内容，OwOh对猫狗分类也不明显；且四
款app都不完全算是社交类，嵌入其他内容占的比重过大。2019年全国城镇宠物犬只数
为5503万只，比2018年增长8.2%；宠物猫只数为4412万只，比2018年增长8.6%
，由此可以看出宠物猫的需求并不比宠物犬小。基于以上两点，猫狗日记的定位为专注猫
狗社区交流。

1.3 需求整理
1.3.1 需求分析
2019年全国城镇宠物犬猫数量达到9915万只，比2018年增长766万只（数据来源：
2019年宠物行业白皮书），由此可见宠物行业发展非常迅速、需求也非常大。
养宠人群基本情况：女性为主，90后是主要养宠群体，未来95后将承担消费主力（养宠
群体趋向女性化和年轻化，数据来源：2019年宠物行业白皮书）。年轻化、女性化也就
意味着养宠群体有更多的可能使用app来分享养宠生活及经验。
随着越来越多的新手开始养宠物，由于缺乏经验，他们会面临一些养宠物的问题。同时，
疫情期间还暴露出一个问题：云吸猫、云吸狗的需求；很大一部分人对养宠物非常感兴趣
但由于时间、环境等原因不能养宠物。因此，用户需要一个专业平台将养宠者、爱好者集
中起来，方便进行经验沟通、分享学习并提供云吸猫、云吸狗的平台。
不了解宠物用品的真实用户体验； 需要了解养宠经验（疾病、粮食、用品等）； 需要一
个集中的云吸猫、云吸狗的平台。
1.3.2 用户定位

缺乏宠物突发问题的处理经验的养宠者 对宠物用品的选择存在问题的养宠者 有分享宠物
生活习惯的养宠者 有意愿养育宠物的用户或有云吸猫、狗的需求的用户

04 产品原型及说明
4.1 登录/注册页面

登录页面

注册页面
未登录状态仅能浏览推荐内容，不能评论、点赞、云领养、发消息、发布动态和问题、分
享等。
（1）页面名称：登录/注册页面
（2）页面入口：未登录状态下，用户发布问题、动态或点击评论、点赞、云领养、发消
息、分享等出现；
（3）页面功能及逻辑交互：
1. 账号登录：
a.第三方授权登录：授权成功后，完成登录，转入猫狗日常—推荐页面。
b.手机验证码登录：初始界面下，呈现手机号码输入栏、验证码输入栏、获取验证码按钮
手机号码默认+86，输入限制11位数字。当输入非数字内容时，输入界面不显示任何内
容；输入数字小于11位时，点击登录提示“请输入11位数字手机号”当输入大于11位
数字时，超过部分不显示。点击获取验证码后，系统自动发验证码短信给输入手机号，验
证码按钮变为60s倒计时，倒计时结束前不可再获取验证码。验证码正确，完成登录，转
入猫狗日常—推荐页面。
c.账号密码登录：输入手机号、密码验证成功后，完成登录，转入猫狗日常—推荐页面。
2.账号注册：在登陆页面点击用户注册转入账号注册页面，可以通过手机号注册或第三方
授权登录，如为首次第三方授权登录，还需验证手机号。操作同账号登录，注册完成，转
入我的资料页面。
3. 忘记密码：通过手机号验证重新修改密码。

4.2 猫狗日常页面

猫狗日常——动态分类——推荐

猫狗日常——动态分类——附近

猫狗日常——动态分类——关注

猫狗日常——动态详情

猫狗日常——评论详情

猫狗日常——发布新动态

猫狗日常——宠物详情（云领养）
（1）页面名称：猫狗日常
（2）页面入口：启动猫狗日记app直接进入“猫狗日常”，或在其它页面时点击底部页
面切换区的“猫狗日常”进入。
（3）页面分布及逻辑交互：
页面切换区：为app主菜单，分为猫狗日常、百科问答、消息、我的，有新内容时在对
应区域出现红点，点击后消失； 动态分类切换区：点击切换或左右滑动切换动态分类，
分为推荐、附近、关注，有新内容时在对应区域出现红点，点击后消失； 动态内容展示
区：展示用户头像、昵称、位置信息以及当条动态的预览(文字、视频、图片)和对应的评
论数、点赞数；点击动态预览内容(文字、视频、图片)进入动态详情，点击评论进入评论
详情，点击用户头像、昵称或云领养进入宠物详情。 动态详情：通过点击动态内容展示
区转入，展示用户头像、昵称、位置信息以及当条动态(文字、视频、图片)和对应的评论
数、点赞数，长按图片可保存，可点赞评论分享举报； 评论详情：通过点击动态内容展
示区或动态详情中评论转入，展示其他用户对当条动态的评论以及评论数，点击回复、评
论框底部呼出键盘，可以回复评论内容和输入评论内容并发送； 宠物详情（云领养）：
通过动态下方的云领养或动态展示区、回答详情和消息详情的头像、昵称点击转入，展示
用户资料及宠物信息，点击图片和头像可以查看大图，可进行举报、关注、发消息、云领
养等操作。 猫狗内容切换：分为仅猫、仅狗、猫狗，点击切换； 发布：点击“+“，发
布新动态。 搜索栏：未点击时出现提示字样（搜索热词），点击搜索栏底部呼出键盘，
通过输入关键字搜索动态内容。

05 总结
以上就是猫狗日记app的产品需求文档。
当前宠物犬、猫养育数量不断上升，且主人逐渐年轻化、女性占据大多数，而自带社交属
性的年轻人们是需要一款专注宠物犬、猫养育的社交app。市面已经存在的类似app都
存在嵌入其他服务而忽略了对社交的本意，所以猫狗日记专注于关于宠物的社交和问答，
为养宠者提供一个交流的平台，为爱宠者提供一个云吸宠的平台。
由于猫狗日记app是社交和问答结合，与市面上存在的宠物社交app存在一定差异，最
有参考价值的竞品应该是微博、百度贴吧、知乎等宠物板块。那么面临的问题是如何将用
户从大平台引入本app，可以从宠物专业性和集中化分享这两方面入手，其后期迭代方向
也应向其靠拢。
