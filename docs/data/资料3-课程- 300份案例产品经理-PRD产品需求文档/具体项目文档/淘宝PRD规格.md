产品吧
产品吧是一个会员自主创建的内容页面。会员可以围绕产品的要求从淘宝网内的资讯，论坛，博客内挑选相关内容组建。

分栏
在产品吧里的某一个区域，这个区域由图片文字链接构成，这块区域的内容用来表达某一个共同的主题。
产品概述及目标
淘宝目前的社区内容结构上来说，已经具有频道，论坛，博客三大产品，其中频道，论坛是针对某一个大的主题来组织内容的，它能满足一般淘宝会员逛的需求，但是对于希望购买某一类具体产品的用户来说用处不大，无法满足他们对特定产品资讯了解的需求，而产品吧这个产品通过用户人工创建，按产品线索查找，可以为这类用户提供特定信息需求和在使用产品过程中的经验分享。
目前垂直行业领域的产品库和本产品比较类似，比如大拿网，网易化妆产品库
这些专业产品库的优点如下： 非常的专业，因为是垂直领域，产品信息全，资料完整，有丰富的图片
缺点：基本上都只集中在某一个具体领域内，比如数码，化妆品。没有一个全面的产品构架，受行业局限性高。另外也无法和交易打通，使用户形成购买。

我们的产品吧，能将淘宝目前的资讯，论坛帖子，博客完整的整合起来，并且通过鼓励用户原创的方式充实博客的内容。另外在接下来的推广过程中，将陆续增加产品的团购，产品清单的推荐，产品买家秀让用户在淘宝能真正获取想要的信息。同时在这个过程中，为淘宝创造信息内容。

PRD 1．管理后台登录
简要说明
淘宝小二登录产品吧的后台管理页面。
业务规则
  只有门户的小二才能进入产品吧的后台管理页面。权限设计能否直接走统一的后台权限控制？
界面原型
 
执行者
   淘宝门户小二
前置条件
   在登录状态下，具有门户小二权限的，可以直接通过链接进入产品吧后台管理页面。
后置条件
    无
主流程 
门户小二直接输入URL
当在未登录状态下，系统显示登录页面。已经登录状态下，进入产品吧后台管理界面，默认显示按类目浏览树下的产品吧。类目浏览树只显示一级大类。

PRD 2．词条管理
简要说明
门户小二可以增加新的产品吧词条。
业务规则
  在产品吧第一期，所有产品吧属于一级分类
界面原型
  

执行者
   门户小二
前置条件
   在词条管理上，点击添加产品吧词条。
后置条件
    添加成功后，关闭当前层，刷新类目列表。
主流程 
门户小二点击“添加产品吧词条”
在当前页面弹出新词条层。 定义如下：
词条名称：输入15个汉字长度的字符串。长度直接在输入框限制，词条需要做唯一性校验（必填项）
分类：小二先选择当前已经有的类目第一级大类。（必填项）
门户小二点击“提交”
系统将内容写入数据库，关闭当前层，刷新分类列表。

 
PRD 3．产品吧列表
简要说明
定义和描述产品吧列表上将显示那些功能和入口
业务规则
  产品吧列表按50条来进行分页。排序规则按时间倒序进行排列。
  在类目树上，第一期只显示第一级大类。
界面原型
 
执行者
   淘宝门户小二
前置条件
   在类目树上选择某一个分类
后置条件
进入搜索结果页面
进入编辑基本信息页面
进入设置吧主层
删除产品吧
主流程 
门户小二输入关键词，点击“搜索”
系统将返回匹配的分类结果列表。列表包含：
匹配产品吧的名称，编辑基本信息，设置吧主，删除
门户小二在类目树里面点击某一个大类
系统显示该大类下的所有产品吧。按50条进行分页
门户小二点击删除
系统弹出提示： “您确认需要删除XXX产品吧？ 是，否。” 
门户小二选择是
系统将删除对应的产品吧，并且刷新当前的类目列表。当在搜索状态下，如果被删除的是单个记录，则显示搜索无结果页面。
当某一个分类上已经设置了吧主，则显示修改吧主

  会员最多只能在三个产品吧里担任吧主，不能超过。
界面原型
 
执行者
   门户小二 
前置条件
   在分类列表中，点击了设置吧主/修改吧主
后置条件
    数据更新后，能立即显示设置成功的吧主。同时系统发送站内信，提醒被设置的吧主产品吧的访问入口地址。
主流程 
门户小二在在分类列表中，点击“设置吧主/修改吧主”
系统弹出设置吧主层，如果当前系统已经设置过吧主，则显示目前的吧主ID
门户小二输入/修改吧主ID，点击提交
系统关闭当前层，返回被修改的分类所在的分类列表页面，假如有吧主，则显示“修改吧主”，假如无吧主，则显示“设置吧主”。
4.1. 系统检查会员是否存在，当小二填写了不存在的会员，系统提示：XXX会员不存在。当某一个会员ID在三个产品吧里担任吧主，系统提示：XXX已经在三个产品吧内担任吧主，不能再担任其他吧的吧主。
