明确新需求或希望改进的产品需求。
明确不需要的变化和注意问题。

2. 用户描述
提示：为了有效地提供满足用户的产品和服务，此部分信息将帮助产品和研发从市场层面理解需求用户价值和必要性。这一部分不是为了说明需求，而是说明我们用户是谁，我们可能为他们解决什么问题。

2.1 用户/市场统计
提示：这里将给出支持产品决策和排序的主要的统计数据。描述定位的目标市场，可能的潜在用户数据或客户销售额收入预期，以及预测的市场大小、增长率。以及市场竞争格局，包括：
您的服务在市场上定位？
你希望功能或服务支持哪些功能？
这些功能或服务如何支持你的目标？

2.2 用户剖析
提示：请在这里描述服务中每个不同的用户。用户的类型可能从权威的专业人士到新手，差别很大。专业人士可能需要一个复杂、灵活的系统，而新手可能只需要够用和易用。因此对目标用户的分析内容包括以下：
该类用户的相关技术背景，比例。
该类用户的主要职责或角色。
该类用户为谁提供服务。
该类用户为服务一般认识和理解。
该类用户相关系统可能面对的主要问题。
该类用户对服务满意的可衡量标准。
该类用户对关键决策的影响（如：使用、购买的决策权）

2.3 用户环境
提示：详细描述目标用户的使用环境，包括以下建议：
用户会在什么条件或环境下产生使用或购买动机。
用户使用服务的主流环境。如：网速、浏览器类型信息。
用户使服务的场所和时间段。如：办公室，工作时间。
用户现在正使什么系统，什么可能促使改变。

2.4 关键用户需求
提示：这里列出用户或客户认为的关键问题或需求，问题描述包括：
问题的原因是什么?
现在怎么解决的？
希望的解决方案是什么？
（此类信息用户帮助理解用户所认为的问题的重要性，将用于需求优先级排序。）
