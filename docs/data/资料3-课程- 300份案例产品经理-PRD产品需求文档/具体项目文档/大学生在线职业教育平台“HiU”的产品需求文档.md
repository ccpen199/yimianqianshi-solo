# 大学生在线职业教育平台“HiU”的产品需求文档

本文档为笔者依据自身和周围用户的切身经历，自己设计的一款简约平台：HiU大学生在
线职业教育平台的PRD，经过前期市场分析以及用户访谈调研，通过报告梳理了用户的
核心需求，并转化为了平台需求，进而将相关资料信息汇总，为平台后续完善和更新提供
依据。

1.2 产品简介
产品类型：HiU大学生在线职业教育平台。为大学生提供高质量指导建议规划，并辅以相
关课程等服务 产品介绍：一款为大学生从初入学到步入职场前，提供高质量的生活职业
规划指导建议的平台。用户可以通过该平台，阅读高质量文章，分享经验，购买职业、兴
趣爱好课程，并方便完成可也，以此不断提升自我。 产品定位：为广大爱好多样、需求
广泛的大学生提供集社区、课业、课程、职业规划指导多方面服务，并且逐渐向各方面深
度挖掘，不断完善平台。


1.3 产品用户
可以看出，在线教育目标用户年龄段集中于20-29，30-39之间，主要以80后、90后为
主。
目标用户为处于不同学校，不同专业，不同规划的大学生用户群。往往为想要明确自我职
业方向，需要指导，并进而提升自身职业能力的职场人士和初入职场的大学生，因此构造
一款产品，为有自学能力的大学生提供基础课程和额外自主学习课程，同时帮助大学生进
入职场加以引导、提供帮助。
而经过笔者自身体验，浏览过部分校园的论坛、微信公众平台投稿等，发现大学生对于教
育方面的需求往往集中在以下几方面：校园教育（基础和专业课程的听讲、作业完成），
职业和兴趣方向的找寻（职业测评、职业分析、性格测评、兴趣引导），继续学业（考研
、出国），找工作（考公、考证、实习和全职）
对于以上几类目标用户，通过简单用户画像和用户场景来进行整理：
小赵同学：大一新生，大学探索阶段，想培养一些额外兴趣爱好，但是不清楚自己适合什
么，应该学什么，去哪学。


用户场景：初入校园，小赵想在网络上培养一些兴趣爱好，并且学习相关的教程资料，但
是网络资源繁杂，难以辨识，浪费时间往往找不到合适的优质课程。 产品期望：通过在
线教育平台，进行个人测评、兴趣爱好推荐和引导，推荐相关的非专业课程，陶冶情操提
升技能。
小刘同学：大二学生，疫情期间，课程较为繁重。
用户场景：疫情期间，小赵同学要在网络上完成一些公共基础课程和专业课程的学习，并
且完成老师布置的课后作业。但是作为大学生，有自己的时间规划，比较排斥直播方式，
想要通过一个平台，能在自己规划的时间，观看课程，并完成上交课后作业。 产品期望
：通过该在线教育平台，查找录播课程，观看后提交课后作业。
小孙同学：大三学生，明确自身应当作出规划，采取一些行动。
用户场景：学校和家庭都暗示自身应当作出规划和选择，但是小孙对选择较为模糊，但不
清楚就业还是考研，自己到底适合找工作还是继续学业，选择了以后该做什么行动。想通
过帮助，明确自身的选择。明确之后，找工作方面：想了解行业发展、企业和行业的模式
、招聘需求、一些学长学姐的经验经历、去哪找实习兼职等。而如果决定继续学业：想了
解国内外高校的水平、考研方面经验和专业课程、书籍。 产品期望：帮助做出职业测评
和规划，并提供相应选择后的优质课程和实时信息。
小林同学：大四学生，已经初步完成未来规划，采取了行动。
用户场景：做出对未来评测和行动后，小林想要继续学习一些职业课程，以此丰富自己的
专业技能，并且想试着传达一些自己的经验和想法给学弟学妹，帮助他们避免弯路。 产
品期望：职业课程，分享职业和规划信息给其他人。

四、平台全局说明
4.1 登录
功能描述：使用用户创建的账号进行登录。
原型：
输入/前置条件：密码或账号任意为空：不允许提交
需求说明：
1.账号、密码匹配验证：不匹配，提示“账号或密码错误”
2.自动登录：
默认“勾选”，“勾选”状态下每次打开客户端软件直接进入客户端 “不勾选”状态下
每次打开客户端需要输入账号密码手动登录
输出/后置条件：


登录成功后，跳转至客户端首页 登录成功后，能以登录状态访问所有页面及Web内嵌页 
登录成功后，客户端顶部栏右上角显示当前账号的“手机号”，没有“手机号”的显示“
用户名”
4.2 注册新账号
功能描述：用户没有账号，可以在平台内点击去网页注册。
原型：
输入/前置条件：手机号不满足11位数字，不允许获取验证码；或验证码不正确，不允许
注册
需求说明：
手机号：不满足11位数字，提示“请输入正确的手机号码” 获取验证码：点击后进入
60s倒数状态，倒数期间，处于禁用状态，60s后，可再次点击 用户名：汉字、字母、数
字、下划线均可，不允许与已有的用户名重复 “不勾选”已经阅读并同意《用户注册协
议》，提示请勾选同意相关协议。
输出/后置条件：


注册成功后，跳转至平台首页 注册成功后，能以登录状态访问所有页面及Web内嵌页 
注册成功后，客户端顶部栏右上角显示当前账号的“手机号”，没有“手机号”的显示“
用户名” 注册未完成，可点击下方跳转会登录页面，凭借已经注册成功的用户名进入登
录状态。
4.3 忘记密码
功能描述：用户如果忘记自己的密码，可在平台通过注册手机号找回。
原型：
输入/前置条件：手机号不满足11位数字，不允许获取验证码；或验证码不正确，不允许
更改密码。
需求说明：
使用已注册的手机号接收验证码，重新输入新密码，提交后直接登录客户端，其中的表单
验证提交时触发：


手机号：为空或不满足11位数字，显示“手机号码有误”；手机号未注册过，显示“此
手机号尚未注册” 验证码：为空或验证码不正确，显示“验证码有误” 设置新密码：为
空或超出6-32个字符范围，显示“密码格式不符，请输入6-32个字符”
输出/后置条件：修改成功后，直接跳转登录页，用户凭借刚修改的密码登录即可。
4.4 退出账号
功能描述：退出平台当前登录的用户账号。
原型：

需求说明：在首页右上角，鼠标移入用户名区域，弹出悬浮列表，点击“退出账号”
后置条件：确认退出后，平台切换至登录页面

五、页面详细说明
5.1 首页

主页共有七个主要功能点，涉及入口的具体功能会在下文中详细说明，此处做总体概述。
主页标签栏
主页标签栏共四个入口。用户可点击课程，进入到课程主页面；点击测评，进入到测评主
页面；点击社区，进入到社区主页面；点击下载APP，用户鼠标移入后，出现悬浮模块，
显示该平台应用的下载二维码。
用户悬浮标签栏
当用户鼠标移动到用户名区域时，悬浮标签栏出现，鼠标移开时，悬浮标签栏消失。悬浮
标签栏中共四个功能点：
个人中心：对用户的个人信息进行集中管理。分为我的课时、购买记录、我的测评、我的
发帖、账户安全。 消息中心：对通知该用户的系统消息进行集中管理。如：课程发布作
业、课程发布资料、账号异常等情况。 学习中心：对用户的学习情况进行统计和管理。
分为：我的课程、错题本、学习资料、课后作业四个入口 退出登录：已登录用户可点击
，点击后退出当前帐号，回到登录首页
课程分类栏
对平台拥有的课程进行分类，并将点击率较高的分类显示在这一栏，用户可点击直接进入
课程首页下的该分类，查看该分类下的课程。
我的课程入口
用户可点击该快捷入口，直接进入学习中心的我的课程页面，对课程进行查看和管理。
课程详情
用户在浏览到感兴趣的课程后，可直接点击该课程封面，进入到课程详情页面。里面包括
课程详情、课程评价。课程整体分为付费和免费课程，依据该项不同，显示页面有差异。
下文会详细介绍。
社区入口

用户可点击进入文章社区，社区内含有高质量经验，专家文章等，分别依据：考证考级、
21级考研、出国留学、就业实习等进行分类。用户点击分类标签，可进入到该分类下查
看文章。用户可以浏览、点赞、发帖，优秀的帖子被管理员加精后，可获得用户活跃值，
出现在社区首页的活跃榜上。
测评入口
用户点击后，进入测评区。测评区依照测评种类分为：就业测评、创业测评、能力测评、
性格测评。用户进入测评并完成后，可在消息中心收到通知，并可进入个人中心查看测评
结果。
5.2 课程
课程主页面可通过上述主页标签栏、课程分类栏、推荐课程栏进入。
课程主页-课程详情交互如下：


用户进入课程主页后，点击感兴趣的课程，可进入到该课程详情界面。分为付费课程和免
费课程。
付费课程显示价格、已购买人数，激发用户购买欲，以及立即付款和试听一节功能。购买
后，再次进入该课程，显示的页面逻辑与免费课程相同。
免费课程可立刻观看课程，并选择添加到我的课程，添加后，用户可在个人中心中查看并
进入。
课程详情页面下滑后，可看见课程简介以及课程评价，为用户提供参考。
播放课程交互：

进入课程详情页面后，点击立即观看课程，可进入播放课程页面。播放课程页面提供视频
、作业、资料、讨论区四个核心功能。
播放课程页面默认停留在视频页面。点击作业后，进入到学习中心的我的作业。点击资料
后，进入到学习中心的我的资料，包含所有我的课程中的可供下载的资料。点击讨论区后
，可查看学习该课程的同学对课程内容的讨论，用户也可以自己发起讨论询问、查看并点
赞他人讨论、搜索关键字讨论。


讨论区中的“返回课程”、学习中心的资料和作业页面，提供所属课程的链接按钮，三个
出口点击后，可回到该课程播放的视频页面。
5.3 个人中心
个人中心可通过所有带主页标签栏头部的页面进入，当鼠标移入到用户的用户名区域时，
悬浮标签栏自动显示，出现个人中心，点击可进入。鼠标移出后，悬浮标签栏自动消失。
个人中心主页交互：


进入个人中心主页后，默认为用户个人中心信息修改页面，用户可在此绑定手机，点击后
，弹出绑定对话框，手机号为11位数字，方可获取验证码，验证码正确可进行手机绑定
。误触绑定手机按钮，可通过对话框的关闭和取消，返回个人中心信息主页。
用户可在此页面，进行用户名、性别、微信号、学校、邮箱的修改。其中：用户名不可与
已有用户重复；性别为单选项。
我的课时交互：
用户在进入个人中心主页后，可通过个人中心左侧标签分类栏，进入到我的课时页面。


该页面包含了所有用户添加的课程，并自动进行课时的计算，显示总课时、已完成课时、
剩余课时，以此激励用户。下方可查看具体课程安排和课时分布，点击继续上课，可对应
进入该课程的视频播放页面。该页面的交互方式和入口与上述相同。
购买记录交互：
用户进入个人中心主页后，可通过个人中心左侧标签分类栏，进入到购买记录页面。该页
面包含用户所有购买记录，包括已完成和未完成。
点击已完成和未完成的tab，下方购买记录相应切换。
已完成包含：课程名称、订单号、订单金额、优惠金额、实付金额、订单时间。
未完成课程包含：课程名称、订单号、订单金额、优惠金额、应付金额、未付金额以及功
能按钮。
功能按钮包括：取消订单和继续付款两种。用户点击取消订单后，该订单取消成功，从购
买记录中删除，点击继续付款，进入到支付宝等第三方付款界面。


我的测评交互：
用户进入个人中心主页后，可通过个人中心左侧标签分类栏，进入到我的测评页面。该页
面包含用户所有的测评记录，包含测评名称，测评时间，以及测评结果，用户可点击查看
测评结果按钮查看。下方为更多测评链接按钮，用户可点击进入到测评主页。
我的发帖交互：


用户进入个人中心主页后，可通过个人中心左侧标签分类栏，进入到我的发帖页面。
此时，该页面左侧标签栏变化，包括：我的帖子、我回复的、我的精品以及返回个人中心
。
我的帖子页面记录所有用户在社区投稿并审核通过发布的帖子，包含用户发帖社区、
发帖名称、发帖时间；我回复的记录用户所有在社区回复的内容，包含回复内容、回复帖
子、回复社区、回复时间；我的精品记录用户在社区投稿、审核通过并被管理员设置为精
品的帖子，该类帖子可在社区主页优先展示，并提升用户活跃度，记录中包含用户发帖社
区、发帖加精名称、加精时间、加精管理员；草稿箱存储用户编辑但是仍未提交的文章。
返回个人中心提供出口，用户可点击，回到个人中心主页。
账号安全交互：
用户进入个人中心主页后，可通过个人中心左侧标签分类栏，进入到账号安全页面。


该页面可帮助用户修改当前登录账户的密码，输入原始密码，核对后，可输入新密码、重
复输入新密码，确认修改后，弹出dialog提示用户设置完成，下次登录时可使用新密码
。
5.4 消息中心
消息中心与个人中心并列，可通过所有带主页标签栏头部的页面进入，当鼠标移入到用户
的用户名区域时，悬浮标签栏自动显示，出现消息中心，点击可进入。鼠标移出后，悬浮
标签栏自动消失。
进入消息中心主页后，可查看到所有系统发给当前用户的消息，进而对通知用户的系统消
息进行集中管理，包含：信息内容、信息时间、指示按钮。信息内容包含：课程发布作业
、课程发布资料、账号异常等情况，
消息后方依据消息类型不同，指向不同出口。如课程发布作业，查看详情后，可进入到学
习中心-我的作业；课程发布资料，查看详情后，可进入到学习中心-我的资料；账号异常
，可点击后方立即咨询，即可弹出官方咨询电话。
5.5 学习中心
学习中心可通过所有带主页标签栏头部的页面进入，当鼠标移入到用户的用户名区域时，
悬浮标签栏自动显示，出现学习中心，点击可进入。鼠标移出后，悬浮标签栏自动消失。


用户进入到学习中心主页面后，左侧由指引的标签栏，共四个核心功能：我的课程、错题
本、
学习资料、课后作业。
顶部显示用户头像、用户名、当前日期、当前天气以及课时信息，包括所有课时、已完成
课时和剩余课时。
我的课程交互：


用户在选择进入学习中心页面后，默认学习中心页面显示左侧标签栏中：我的课程页面。
该页面包含所有已添加的课程，用户可点击右侧按钮继续学习或开始学习，点击后，进入
该课程的视频播放页面。右侧按钮下方，共三个核心功能入口：错题本、学习资料、课后
作业。与左侧标签栏入口相同。
错题本交互：
用户在学习中心左侧标签栏或我的课程下方入口，两处均可进入到错题本页面。
错题本中包含用户所有所做的课业中的错题，可供用户二次复习使用。当鼠标移入到课程
封面时，课程页面变换为右图所示，出现查看错题按钮，用户点击后，即可进入到该课程
的对应错题本查看错题。当鼠标移出后，封面恢复为左图所示。
此外，用户可通过搜索，快速找到所需要的课程对应的错题本。
学习资料与课后作业交互：


用户在学习中心左侧标签栏，可进入到学习资料、课程作业入口。
学习资料页面：记录当前登录用户已添加课程的所有可供下载的资料，用户可通过搜索栏
搜索关键字，快速找寻课程的学习资料。
学习资料包含：学习资料名称、来自的对应课程、上传时间、文件大小和下载按钮。点击
下载后，即可快速下载该学习资料。 同时，学习资料所来自的对应课程为链接按钮，点
击后可进入到该课程的视频播放页面，便于用户及时查看回忆相应课程章节。
课程作业页面：记录当前登录用户已添加课程的所有课程作业，用户可通过搜索栏搜索关
键字，快速找寻课程的课程作业。
课程作业信息包含：课程作业名称、布置时间、截止时间、完成状态、功能按钮和查看该
课程。其中功能按钮根据作业的完成状态而变化：当作业为未完成状态，且未到截止日期
，用户可点击“做作业”功能按钮进入到完成作业页面。当作业为已完成状态，且未到截
止日期，用户可点击“修改”功能按钮，进入到完成作业页面。 当课程作业到达截至时
间后，功能按钮失效，用户无法修改和完成，便于老师及时有效批改并给予反馈，以此创


建错题本。 课程作业提供“查看该课程”的链接按钮，用户点击后，可进入到相应课程
的播放界面，便于用户及时回忆、复习对应课程。
5.6 测评主页
测评主页核心功能点有两处：
测评中心左侧的标签分类栏：分为就业测评、创业测评、能力测评和性格测评。用户通过
点击不同的标签分类，主页面的测评内容更换。
测评信息：每个测评包含如下信息：测评名称、测评简介、测评题目量、测评平均用时、
测评完成人数以及开始测评按钮。其中测评题目量、测评平均用时可辅助用户决策是否当
下立刻完成测评，测评完成人数提升用户参与度。
当完成测评后，测评结果需要过一段时间生成，生成后，系统会发送信息到该用户的消息
中心，用户点击消息中心后，受到测评结果信息，通过查看详情，即可查看测评结果。


5.7 社区主页
社区页面核心功能点有两处：分类标签栏以及编辑。
分类标签栏：可依据诸如考证考级、出国留学、就业实习、考公务员等的不同分类，展示
不同分类下的精品文章，便于用户快速定位。
编辑：用户可通过此功能，撰写经验或精品文章，投稿后经由管理员审核，可发布在社区
，并提升活跃值，活跃值靠前的，可出现在当日活跃榜上。
分类标签栏交互：


各大分类标签点击后，下方文章处页面变化，显示的页面内容，与社区主页当前显示的页
面内容形式基本相同。包括：文章标题、文章简介、文章撰写人、文章发表时间、文章点
赞人数。
其中我的订阅略有不同：在用户对某一社区感兴趣后，即可关注该社区。此后，该社区出
现的最新精品文章也显示在我的订阅页面下。其中显示订阅人或订阅社区名称、文章标题
、文章第一段、文章发表时间、文章点赞人数，并且将会突出显示订阅人或订阅社区头像
，以类似于“回复”的样式出现，提升用户对订阅主题的信息确认，增加用户对该社区的
粘性和活跃度。
编辑交互：


社区主页顶部分类栏目与其他首页顶部分类栏目略有不同，右侧增加编辑按钮。
点击编辑按钮后，用户可进入编辑文章页面，在此处可以撰写文章分享经验，当编辑文章
后，可点击“提交审核”后，交给管理员审核文章，审核通过后，系统将发送消息到该用
户的消息中心处通知，同时该用户个人中心的我的发帖一栏，“我的帖子”处同样包含审
核通过的帖子。
用户也可点击“保存文章”，此时用户编辑的文本将存入草稿箱，用户查看草稿箱后，可
通过“继续编辑”，进入编辑文章页面，再次编辑未完成的文章。
