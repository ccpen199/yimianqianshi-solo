2. 撞库准入流程
1）省呗发起用户信息校验
用户点击****的省呗广告进行申请，省呗向市场推广系统发起用户信息校验请求，校验接
口传递MD5后的手机号，并附带签名，签名规则详见省呗联合注册接口文档v1.4的【5.
】。
2）判断用户类型
市场推广收到校验请求后进入撞库准入流程，根据传入的MD5手机号，查询该用户是否
在***动用过：未动用过的话撞库准入通过（默认未动用过的用户撞库准入通过，可更改
配置为不通过），动用过的话进入用户风控等级验证流程。
3）用户风控等级验证
若用户的风控等级为A、B、C、D、E、Z，则验证结果为通过（默认全部风控等级的用户

空或获取失败，则验证结果为通过。

3. 撞库处理流程
1）动用过****的用户判断
用户信息通过撞库准入后，市场推广会进行撞库处理，查询判断用户是否在****动用过。
如果从未动用过，则进入风控4个黑名单库验证流程；如果动用过，则进一步判断该用户
上一次在****结清的时间。
2）用户上一次结清时间判断
如果距离用户上一次在****结清的时间不超过30天(天数可配置)或尚未结清，则市场推广
得到撞库结果为老用户；如果距离用户上一次结清超过了30天(天数可配置)，则同样进
入风控4个黑名单库验证流程。


3）风控4个黑名单库验证
进入黑名单库验证流程的用户包括：从未在****动用过的用户和上一次在****结清的时间
超过30天(天数可配置)的用户。市场推广调用风控接口判断是否命中风控4个黑名单，
若有任何1个命中，则市场推广得到撞库结果为不通过；若未命中任何1个名单库，则市
场推广得到撞库结果为新用户、进入联合注册流程。

4. 补充
出于节省资源考虑，市场推广只记录最近2周的撞库联登数据，超过2周的数据定期删除
。
四、其他说明
风控、会员中心接口调用失败需重新发出请求，连续3次失败报警通知到相关责任人（开
发，测试，产品，运营）。 调用风控接口查询用户的风控等级，每天的调用上限为4万
人，预估后期要支持每天40万撞库查询量。 测试密钥用**********，与省呗测试联调过
程中需要提供校验接口url，注册接口url。 产品会提供邮箱给省呗接收线上密钥，上生
产时会提供密钥给开发测试，同时需要准备好线上校验接口url，注册接口url，下载url

（新、老用户）。 接口约定、加密、签名、接口详细定义详见附件《省呗联合注册接口
文档v1.4》。
